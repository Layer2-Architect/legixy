# ux-perspectives.md（UI/UX 層・操作モデル固有観点ベース）

このファイルは TP 生成時に AI が参照する観点ナレッジベース（UI/UX 層用）。
既存ツール（同領域の代表的アプリケーション）から抽出した操作モデル観点を蓄積する。

> **このファイルは雛形である。** プロジェクトに導入後、領域固有の操作モデル観点を追記すること。
> AT で発見された違和感・つまずきポイントは必ずここに昇格（→ `07-at-and-nfr.md` §2）。

---

## 使い方

- UI / UX 領域の TP 生成時に AI が参照
- AT で発見された違和感・つまずきポイントは必ずここに昇格
- 観点の出所（参考にした既存ツール）を明示

---

## 汎用 UI / UX 観点

### 入力

- [ ] 入力長制限の境界での挙動・フィードバック
- [ ] 不正入力の即時 vs 遅延フィードバック
- [ ] IME 確定中の操作受付
- [ ] ペースト時の改行・特殊文字の扱い
- [ ] 自動補完の介入と取消

### 状態の可視化

- [ ] 編集中・保存中・保存済み状態の区別
- [ ] ロック状態の visual feedback
- [ ] 処理中（バックグラウンド処理・AI 検証中等）の進捗表示と中断可否
- [ ] 不可逆操作の事前警告
- [ ] エラー状態の表示と回復手段の提示

### 操作の取り消し

- [ ] Undo の境界（どこから先は Undo できないか）
- [ ] Redo の境界
- [ ] Undo スタックがリセットされる条件
- [ ] 外部保存後の Undo 可否
- [ ] 連続操作の merge（細かい操作を 1 つの undo 単位にまとめる）

### キーボード / マウス / タッチ

- [ ] ショートカットキーの一貫性（同一機能は同一キー）
- [ ] modifier key の挙動
- [ ] drag 中のキャンセル方法
- [ ] context menu と toolbar の機能対応
- [ ] long-press / right-click の役割分担

### フォーカスと選択

- [ ] フォーカス遷移の予測可能性
- [ ] 選択状態の永続化（操作後・保存後・再起動後）
- [ ] 複数選択時の操作の意味
- [ ] フォーカストラップ（モーダルダイアログ等）

### ナビゲーション

- [ ] 戻る操作の到達点が予測可能か
- [ ] ディープリンク・bookmarkability
- [ ] 履歴の意味（ブラウザ履歴 / アプリ内履歴の区別）

---

## エラー・例外の UX

- [ ] エラーメッセージから「何が起きたか」「どうすれば良いか」が読み取れるか
- [ ] 復旧不能エラーと復旧可能エラーの区別
- [ ] バルク操作で一部失敗時の表示（成功 N 件 / 失敗 M 件 / 詳細）
- [ ] オフライン時の挙動
- [ ] サーバ側エラーのユーザ表現

---

## 待機 / 進捗の UX

- [ ] ある時間以上の処理に進捗表示
- [ ] 処理時間の見積もり（残り時間表示）
- [ ] cancel 可否
- [ ] 完了通知（フォアグラウンド / バックグラウンド）

---

## 永続化と同期

- [ ] 自動保存の頻度と粒度
- [ ] クラッシュリカバリ（再起動時の復元）
- [ ] 複数デバイス間の同期（必要な場合）
- [ ] オフライン編集と再接続時の merge

---

## アクセシビリティ

- [ ] キーボードのみでの全操作可能性
- [ ] スクリーンリーダー対応（aria-* / 役割ラベル）
- [ ] フォーカス可視性
- [ ] 色覚多様性対応（色だけに頼らない情報伝達）
- [ ] フォントサイズ拡大対応
- [ ] アニメーション軽減オプション（prefers-reduced-motion）

## 領域固有観点（ここから下はプロジェクトに応じて追記）

### 例: タイムライン編集系（DAW 風）

#### Track 操作
- [ ] track 追加時のデフォルト位置
- [ ] track 削除確認（中身がある場合）
- [ ] track 並べ替えの drag feedback
- [ ] 最後の 1 track の削除可否
- [ ] mute / solo の相互排他性

#### Timeline 操作
- [ ] zoom レベルの記憶・復元
- [ ] horizontal / vertical スクロールの慣性
- [ ] playhead と編集 cursor の関係
- [ ] イベントの drag 時の snap

#### Region / Event の編集
- [ ] 重なるイベントの選択 priority
- [ ] copy / paste 時のトラック対応
- [ ] cut の semantics（削除？切り取り？）
- [ ] 複数選択中の操作の集約

### 例: バージョン管理系（Git 由来）

#### 履歴操作
- [ ] 履歴ビューと現在状態の対応
- [ ] 過去バージョンの参照と編集の区別
- [ ] 分岐・マージの可視化
- [ ] 差分表示の粒度

#### 競合・ロック
- [ ] 他者ロック中の閲覧可否
- [ ] ロック解除の権限
- [ ] 競合検出のタイミング

### 例: プレビュー / モード切替系

- [ ] 編集モードと preview モードの遷移
- [ ] モード遷移時の作業中データ保護
- [ ] preview の同期性（編集が即時反映されるか）

### 例: 複数視点・複数表示

- [ ] 同一データを複数視点で同時編集できるか
- [ ] 視点間の同期方針
