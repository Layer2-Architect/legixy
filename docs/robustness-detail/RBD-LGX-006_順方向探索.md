
### Entity クラス

```

─────────
- ノード : ノードのコレクション

```

```
走査結果
─────────
- 到達ノード : ノード識別子のコレクション
```

```
打ち切り情報
─────────
```

## 2. クラス間の関係

| 関係元 | 関係先 | 種類 | カーディナリティ | 説明 |
|---|---|---|---|---|
| 走査結果 | 打ち切り情報 | aggregation | 0 対 1 | 走査結果は打ち切り情報を保持する（最大深度指定時のみ生成） |
| 有向グラフ | ノード / エッジ | composition | 1 対多 | 有向グラフはノード・エッジを所有する |
| 順方向探索統括処理 | グラフ読み込み処理 / 起点確認処理 / BFS走査処理 / 走査結果整形処理 | 依存 | — | 統括処理が各処理を協調利用する |
| グラフ読み込み処理 | グラフ定義境界 | 依存 | — | グラフ定義を読み込む |
| グラフ読み込み処理 | 有向グラフ | 生成 | 1 対 1 | グラフ定義から有向グラフを構築する |
| 起点確認処理 | 有向グラフ | 依存 | — | 起点ノードの存在を照合する |
| BFS走査処理 | 有向グラフ | 依存 | — | 下流エッジを順方向に辿る |
| BFS走査処理 | 走査結果 | 生成 | 1 対 1 | 到達ノードと深度を走査結果に記録する |
| BFS走査処理 | 打ち切り情報 | 生成 | 0 対 1 | 深度超過ノードが存在する場合のみ打ち切り情報を生成する |
| 走査結果整形処理 | 走査結果 | 依存 | — | 走査結果を読み整形する |
| 走査結果整形処理 | 打ち切り情報 | 依存 | — | 打ち切り情報が存在する場合に読む |
| 走査結果整形処理 | 走査結果出力窓口 | 依存 | — | 整形済み走査結果を出力窓口へ渡す |

## 3. ステレオタイプ凡例

- `<<boundary>>`: アクター・外部ファイル / 保存域との境界
- `<<control>>`: 探索フローの制御ロジック
- 有向グラフ / 走査結果 / 打ち切り情報 は実行時に構築・破棄される一時データ（永続化対象ではない。読み取り専用操作の出力）
