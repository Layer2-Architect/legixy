# RBD-LGX-003: サブノード自動抽出 のクラス図

**親 SEQA**: SEQA-LGX-003
**親 RBA**: RBA-LGX-003 / **親 UC**: UC-LGX-003
**レイヤ**: 具体側（クラス図レベル、言語非依存）

> **記述規律**: RBA-LGX-003 のドメイン主語をクラスに mapping し、クラス境界・属性（概念型）・操作（**操作名は人間の言語**）を識別する。関数名（言語命名規則）・引数具体型・戻り型・修飾子・言語固有メカニズムは書かない（DD で初出）。`04-iconix-layer.md` §5。
>
> **ハードルール 10**: 言語固有のジェネリック型・並行修飾子・公開修飾子・モジュール識別子・命名規則に従う関数呼び出しが混入したら違反。`scripts/trace-check.sh` [5/5] が検出する。属性・戻りの「識別子」「数値」「真偽」「文字列」「パス」「バージョン」「〜種別」「〜への参照」「〜のコレクション」は**概念型**（言語非依存）で、言語固有型は DD で確定する。

## 2. クラス間の関係

| 関係元 | 関係先 | 種類 | カーディナリティ | 説明 |
|---|---|---|---|---|
| 有向グラフ | ドキュメントノード / サブノード / エッジ | composition | 1 対多 | 有向グラフはノード・サブノード・エッジを所有する |
| 自動生成サブノード | 見出し階層パス | aggregation | 多対 1 | 自動生成サブノードは見出し階層パスに由来する |
| 自動生成サブノード | ドキュメントノード | 依存 | 多対 1 | 親ドキュメントノードを参照する |
| 明示サブノード | ドキュメントノード | 依存 | 多対 1 | 親ドキュメントノードを参照する |
| サブノード抽出統括処理 | 見出し抽出処理 / サブノード ID 生成処理 / サブノードマージ処理 | 依存 | — | 統括処理が各処理を協調利用する |
| サブノード抽出統括処理 | グラフ定義境界 | 依存 | — | グラフ定義と明示サブノード定義を読む |
| 見出し抽出処理 | ドキュメントファイル境界 | 依存 | — | ドキュメントファイルを読む |
| 見出し抽出処理 | 見出しテキスト正規化処理 | 依存 | — | 見出しテキストの正規化を依頼する |
| 見出し抽出処理 | 見出し階層パス | 生成 | 1 対多 | 抽出した見出しから見出し階層パスを生成する |
| 見出しテキスト正規化処理 | 見出し階層パス | 依存 | — | 見出し階層パスの各テキストを正規化し確定する |
| サブノード ID 生成処理 | 見出し階層パス / ドキュメントノード | 依存 | — | 階層パスと親 ID からハッシュ対象文字列を構築する |
| サブノード ID 生成処理 | 自動生成サブノード | 生成 | 1 対 1 | ハッシュ計算（先頭 16 文字）でサブノード ID を確定し生成する |
| サブノードマージ処理 | 自動生成サブノード / 明示サブノード / 有向グラフ | 依存 | — | 衝突解決後に確定済みサブノードとエッジを有向グラフへ反映する |

## 3. ステレオタイプ凡例

- `<<boundary>>`: 外部ファイル・保存域との境界
- `<<control>>`: フローの制御・変換ロジック
- `<<persistent>>`: 永続保存域に由来するデータ（有向グラフ上のノードとして永続化される）
- ドキュメントノード / 見出し階層パス / 有向グラフ は実行時に構築・参照されるデータ（ノード永続化は有向グラフが担う）
