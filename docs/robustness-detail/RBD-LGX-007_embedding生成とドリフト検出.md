# RBD-LGX-007: embedding 生成とドリフト検出 のクラス図

**親 SEQA**: SEQA-LGX-007
**親 RBA**: RBA-LGX-007 / **親 UC**: UC-LGX-007
**レイヤ**: 具体側（クラス図レベル、言語非依存）

> **記述規律**: RBA-LGX-007 のドメイン主語をクラスに mapping し、クラス境界・属性（概念型）・操作（**操作名は人間の言語**）を識別する。関数名（言語命名規則）・引数具体型・戻り型・修飾子・言語固有メカニズムは書かない（DD で初出）。`04-iconix-layer.md` §5。
>
> **ハードルール 10**: 言語固有のジェネリック型・並行修飾子・公開修飾子・モジュール識別子・命名規則に従う関数呼び出しが混入したら違反。`scripts/trace-check.sh` [5/5] が検出する。属性・戻りの「識別子」「数値」「真偽」「〜への参照」「〜のコレクション」は**概念型**（言語非依存）で、言語固有型は DD で確定する。

## 2. クラス間の関係

| 関係元 | 関係先 | 種類 | カーディナリティ | 説明 |
|---|---|---|---|---|
| 生成結果集計 | エラー詳細 | composition | 1 対多 | 生成結果集計はエラー詳細を所有する |
| 対象ノード一覧 | ノード | composition | 1 対多 | 対象ノード一覧はノードを所有する |
| embedding 生成統括処理 | ノード一覧取得処理 / ハッシュ照合処理 / 前処理適用処理 / 推論処理 / embedding 格納処理 / サブノード走査処理 | 依存 | — | 統括処理が各処理を協調利用する |
| ノード一覧取得処理 | グラフ定義境界 / 対象ノード一覧 | 依存 | — | グラフ定義を読み込み対象ノード一覧を確定する |
| ハッシュ照合処理 | 成果物ファイル境界 / embedding 格納境界 / ノードコンテンツ / ハッシュ照合結果 | 依存 | — | ファイル本文と既存ハッシュを照合し照合結果を確定する |
| 前処理適用処理 | ノードコンテンツ / 前処理済みコンテンツ | 依存 | — | ノードコンテンツを受け取り前処理済みコンテンツを確定する |
| 推論処理 | ONNX モデル境界 / 前処理済みコンテンツ / embedding ベクトル | 依存 | — | モデルを参照し前処理済みコンテンツから embedding ベクトルを生成する |
| embedding 格納処理 | embedding ベクトル / embedding 格納境界 | 依存 | — | embedding ベクトルとハッシュとバージョンを格納境界に書き込む |
| サブノード走査処理 | 対象ノード一覧 / embedding 生成統括処理 | 依存 | — | コンテンツ範囲を切り出しサブノード単位で統括処理に委譲する |
| ノード一覧取得処理 | 対象ノード一覧 | 生成 | 1 対 1 | グラフ定義から対象ノード一覧を確定する |
| ハッシュ照合処理 | ハッシュ照合結果 | 生成 | 1 対 1 | 照合を経てハッシュ照合結果（3 状態）を確定する |
| 推論処理 | embedding ベクトル | 生成 | 1 対 1 | 推論を経て embedding ベクトルを生成する |

## 3. ステレオタイプ凡例

- `<<boundary>>`: アクター・外部ファイル / 保存域との境界
- `<<control>>`: embedding 生成フローの制御ロジック
- `<<persistent>>`: 永続保存域に由来するデータ（embedding ベクトルは格納境界に書き込む・前処理済みコンテンツはノード処理中の中間データ）
- 対象ノード一覧・ノードコンテンツ・ハッシュ照合結果・生成結果集計は実行時に構築・破棄される一時データ（永続化対象ではない）
