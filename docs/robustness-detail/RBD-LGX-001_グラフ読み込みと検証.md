# RBD-LGX-001: グラフ読み込みと検証 のクラス図

**親 SEQA**: SEQA-LGX-001
**親 RBA**: RBA-LGX-001 / **親 UC**: UC-LGX-001
**レイヤ**: 具体側（クラス図レベル、言語非依存）

> **記述規律**: RBA-LGX-001 のドメイン主語をクラスに mapping し、クラス境界・属性（概念型）・操作（**操作名は人間の言語**）を識別する。関数名（言語命名規則）・引数具体型・戻り型・修飾子・言語固有メカニズムは書かない（DD で初出）。`04-iconix-layer.md` §5。
>
> **ハードルール 10**: 言語固有のジェネリック型・並行修飾子・公開修飾子・モジュール識別子・命名規則に従う関数呼び出しが混入したら違反。`scripts/trace-check.sh` [5/5] が検出する。属性・戻りの「識別子」「数値」「真偽」「〜への参照」「〜のコレクション」は**概念型**（言語非依存）で、言語固有型は DD で確定する。

## 2. クラス間の関係

| 関係元 | 関係先 | 種類 | カーディナリティ | 説明 |
|---|---|---|---|---|
| 検証報告 | 検証所見 | composition | 1 対多 | 検証報告は検証所見を所有する |
| 有向グラフ | ノード / エッジ | composition | 1 対多 | 有向グラフはノード・エッジを所有する |
| 検証統括処理 | 設定解決処理 / グラフ構築処理 / 形式検証処理 / 意味整合検証処理 / 検証結果集約処理 | 依存 | — | 統括処理が各処理を協調利用する |
| 設定解決処理 | 設定ファイル境界 | 依存 | — | 設定を読み込む |
| グラフ構築処理 | グラフ定義境界 / 成果物ファイル境界 | 依存 | — | グラフ定義・成果物を読み込む |
| 形式検証処理 | 有向グラフ / 検証設定 | 依存 | — | グラフと設定を照合する |
| 意味整合検証処理 | 意味ベクトル境界 / 意味ベクトル | 依存 | — | 意味ベクトルを参照する |
| 検証結果集約処理 | 検証所見 / 検証報告 / 検証結果出力窓口 | 依存 | — | 所見を集約し報告を出力へ渡す |
| 設定解決処理 | 検証設定 | 生成 | 1 対 1 | 設定内容から検証設定を確定する |

## 3. ステレオタイプ凡例

- `<<boundary>>`: アクター・外部ファイル / 保存域との境界
- `<<control>>`: 検証フローの制御ロジック
- `<<persistent>>`: 永続保存域に由来するデータ（意味ベクトルは保存域から読み出す）
- 検証設定 / 有向グラフ / 検証所見 / 検証報告 は実行時に構築・破棄される一時データ（永続化対象ではない。read-only 判定の出力）
