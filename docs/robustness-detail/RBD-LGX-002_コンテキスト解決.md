# RBD-LGX-002: コンテキスト解決 のクラス図

**親 SEQA**: SEQA-LGX-002
**親 RBA**: RBA-LGX-002 / **親 UC**: UC-LGX-002
**レイヤ**: 具体側（クラス図レベル、言語非依存）

> **記述規律**: RBA-LGX-002 のドメイン主語をクラスに mapping し、クラス境界・属性（概念型）・操作（**操作名は人間の言語**）を識別する。関数名（言語命名規則）・引数具体型・戻り型・修飾子・言語固有メカニズムは書かない（DD で初出）。`04-iconix-layer.md` §5。
>
> **ハードルール 10**: 言語固有のジェネリック型・並行修飾子・公開修飾子・モジュール識別子・命名規則に従う関数呼び出しが混入したら違反。`scripts/trace-check.sh` [5/5] が検出する。属性・戻りの「識別子」「数値」「真偽」「〜への参照」「〜のコレクション」は**概念型**（言語非依存）で、言語固有型は DD で確定する。

## 3. ステレオタイプ凡例

- `<<boundary>>`: アクター・外部ファイル / 保存域との境界
- `<<control>>`: コンテキスト解決フローの制御ロジック
- 解決済み起点 / 有向グラフ / 上流成果物一覧 / コンテキスト解決結果 は実行時に構築・破棄される一時データ（永続化対象ではない）
- 監査ログ境界の書込先（engine.db 上）は `<<persistent>>` 保存域に分類されるが、Entity としての表現は監査ログ境界が担う
