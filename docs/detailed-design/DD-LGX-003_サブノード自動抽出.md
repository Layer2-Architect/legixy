# DD-LGX-003: サブノード自動抽出の詳細設計

**親 SEQD**: SEQD-LGX-003
**親 RBD**: RBD-LGX-003 / **親 UC**: UC-LGX-003
**対象言語**: Rust（legixy-graph crate 内）
**境界 API 凍結**: yes（本ドキュメント確定後、crate 間公開 API は変更不可。追加は許容、削除・改名・型変更は次版 SPEC 改訂。HR7）

> 言語別規律は `guides/language-stacks/rust.md`。crate 境界・共有型は **ADR-LGX-020** で凍結済（本 DD は参照する）。型・アルゴリズムは v3 実装（traceability-engine.v3 `crates/lx-graph/src/subnode/`）を底本とし引数互換・ID 生成互換を保つ。

## 1. 対象範囲

- **主 crate**: `legixy-graph`（サブノード ID 生成・見出し正規化・見出し抽出・ParentChild エッジ生成・グラフパース統合）
- **依存 crate（共有型は ADR-LGX-020、再定義しない）**: `legixy-core`（`Id` / 共通エラー）。サブノード自動抽出はグラフ読み込み（UC-LGX-001）の内部処理として実行されるため、上位呼び出しは `legixy-check` / `legixy-ctx` / `legixy-cli` が `legixy-graph` の `parse_graph` を呼ぶことで間接的に本 UC を起動する
- **公開 API surface**: 本 DD §3（`legixy-graph` crate の公開関数群。サブノード抽出・ID 生成・正規化・判定の各関数）
- **関連 SEQD**: SEQD-LGX-003

## 2. 型定義

### 2.1 主要データ型

```rust
// legixy-graph（共有型は ADR-LGX-020。greph 全体は UC-LGX-001 の DD-LGX-001 で定義済み。
// 以下は本 UC で新規確定する型のみ記載。TraceGraph / Node / Edge は再定義しない）

// 自動生成サブノードの中間表現（抽出段階でのみ使用し、最終的に Node へ変換される）
pub struct AutoSubnode {
    pub id: String,                   // "{parent_id}#{hex16}" 形式（SUBNODE-INV-5/6）
    pub parent_id: String,            // 親ドキュメントノードの NodeId
    pub anchor: String,               // 正規化前の生見出しテキスト（# マーカー・closing # 除去済、前後 trim 済）
    pub heading_path: Vec<String>,    // 正規化済み見出し階層パス（h2: 1要素、h3: [h2, h3] の 2要素）
    pub content_range: (usize, usize), // 本文のバイト半開区間 [body_start, end_byte)
}

// 見出し抽出の内部作業型（pub ではない）
struct PendingHeading {
    level: u8,
    raw_line: String,
    normalized: String,
    line_start: usize,   // 行頭バイトオフセット（trim前）
    body_start: usize,   // 見出し行の次の行頭バイトオフセット
    h2_context: Option<String>, // h3 の場合の直上 h2 正規化テキスト
}

// Node, Edge, EdgeKind, SubnodeKind, UnresolvedEdge, TraceGraph は ADR-LGX-020 / DD-LGX-001 参照
// NodeId = String（legixy-graph の型エイリアス）
```

### 2.3 エラー型

```rust
// legixy-graph（パース実行時失敗 = exit 1。自動抽出の部分失敗は Err ではなくスキップ）
pub enum GraphParseError {
    Io(std::io::Error),          // graph.toml / Markdown ファイル読込 I/O 失敗（?演算子で伝播）
    TomlParse(toml::de::Error),  // graph.toml の TOML 構文不正
    ValidationError(String),      // 不正 edge kind / 絶対パス / ..セグメント（セキュリティ検証失敗）
}
```

- 自動生成サブノード抽出時のドキュメントファイル不存在・読込失敗は `GraphParseError` に昇格させず**スキップして継続**（部分失敗トレランス、NFR REL.02、SPEC-LGX-002.REQ.10、SEQD-LGX-003 代替 2a）。
- 不正 edge kind・パス逸脱は `ValidationError` として即時 `Err` 返却（`parse_graph` が `Err(GraphParseError)` を返し、呼び出し側が exit 1 へ変換）。
- 終了コード変換は `legixy-cli` / `legixy-check` の責務（LGX-COMPAT-001 §3）。

## 5. ライフタイム / 所有権 / 借用 方針

- `parse_graph` / `parse_graph_from_str` は `&Path` / `&str` を借用し、`TraceGraph`（所有権）を返す。呼び出し側が所有権を保持し、下流 crate（check/ctx/nav）は `&TraceGraph` で借用する（DD-LGX-001 §5 と同方針）。
- `extract_subnodes` / `extract_subnodes_with_levels` は `&str`（コンテンツ）を借用し、`Vec<AutoSubnode>`（所有権）を返す。`AutoSubnode` の各 `String` フィールドは個別所有（参照保持なし）。
- `generate_subnode_id` は `&str` + `&[String]` を借用し `String`（所有権）を返す。
- `normalize_heading` は `&str` を借用し `String`（所有権）を返す。
- `TraceGraph` 内の `IndexMap<NodeId, Node>` はノードを所有する（`Node` の `String` フィールドも所有）。隣接リスト（adjacency_fwd/rev）は `HashMap<NodeId, Vec<usize>>` でエッジインデックスを保持（所有）。
- `Arc` / `Mutex` 不要（サブノード抽出は単一スレッド逐次、§7）。ライフタイム注釈は `'static` バウンドを要求しない（呼び出しスコープ内完結）。
- `PendingHeading` は `extractor.rs` 内部の一時所有構造体。`extract_subnodes_with_levels` の返却後は解放される。

### 5.1 正規化の適用順序（v3 `normalizer.rs` 実測底本、SPEC-LGX-002.REQ.06 の実装）

```rust
// normalize_heading() の処理順（順序を変えると ID が変わるため変更禁止）
// 1. Markdown 装飾除去（長いマーカー優先: ** / __ / ~~ → * / _ / `）
// 2. Unicode NFC 正規化（unicode-normalization crate の .nfc()）
// 3. U+3000（全角スペース）→ ASCII スペース
// 4. 前後トリム（.trim()）
// 5. 連続 whitespace → 単一 ASCII スペース（is_whitespace() で判定、スペース・タブ等全種別対象）
```

## 6. エラー伝播戦略

- **グラフパース失敗（exit 1）**: `parse_graph` は `GraphParseError::Io` / `TomlParse` / `ValidationError` のいずれかを `Err` で返す。呼び出し側 (`legixy-check` / `legixy-cli`) が `CheckError::GraphLoad` / 終了コード変換を担う（DD-LGX-001 §6 と同方針）。
- **部分失敗継続（ドキュメントファイル不存在・読込失敗）**: `parser.rs` の自動抽出ループ内でファイル不在は `continue`、read 失敗は `match { Err(_) => continue }` で黙殺し残りのドキュメントノードの処理を続行する（NFR REL.02、SPEC-LGX-002.REQ.10、v3 `parser.rs:109-112` 実測底本）。呼び出し側に partial failure の通知は行わない（グラフ全体パースが成功扱いで `Ok(TraceGraph)` を返す）。
- **明示サブノードとの ID 衝突・自動同士の縮退**: `parser.rs` の挿入処理で衝突した自動生成ノードを `continue`（スキップ）する。エラー・Warning は発しない（生成段階での静粛処理。SPEC-LGX-002.REQ.12 / SEQD-LGX-003 代替 3a / 例外フロー）。縮退は `IndexMap::insert` の後勝ちセマンティクスにより自然実現（v3 `parser.rs:126-145` 実測底本、S2-15 後勝ち縮退）。衝突検出・SubnodeIdCollision Warning 発行は `legixy-check` が担う（SPEC-LGX-004.REQ.14）。
- **不正 edge kind**: `GraphParseError::ValidationError` として即時 `Err` 返却（SPEC-LGX-002.REQ.10 の「クラッシュ禁止」は維持、通常エラー経路）。
- **パス逸脱（絶対パス / `..` セグメント）**: `validate_path_scope` が `GraphParseError::ValidationError` を返す（NFR SEC.06）。
- **panic 禁止**: `subnode/` 各モジュールの全関数で `unwrap()` / `expect()` の直接使用を禁止（`rust.md §4`）。`validate_explicit_name` の `unwrap()` は `name.is_empty()` チェック後に呼ぶため安全だが、実装では `chars().next()` ガード後に実施（v3 実測どおり）。

## 7. 並行性 / 非同期境界

- サブノード自動抽出は `parse_graph` の内部処理として**同期・単一スレッド・逐次**で実行される。async なし。
- 各ドキュメントノードの見出し抽出（`extract_subnodes_with_levels`）は逐次ループで処理する。ドキュメント間の並列化は将来最適化（NFR PERF.04/05 を逐次で充足見込み、benches で測定）。
- 外部ファイル更新との並行整合性は射程外（NFR REL.04 の観点、TP-LGX-004 関連）。

| 分類 | 内容 | 対応 TP |

| Integration: パース統合 | `parse_graph_from_str` でのドキュメントノード・自動生成サブノード・明示サブノード・ParentChild エッジの統合生成。縮退（後勝ち）・明示優先（スキップ）・部分失敗継続 | TP-LGX-004 |
