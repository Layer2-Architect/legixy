# ADR-LGX-002: embed `--node`/`--force` の凍結契約への加算的拡張

**ステータス**: accepted
**起票日**: 2026-06-10
**承認日**: 2026-06-10
**承認者**: 開発者（人間裁定、AskUserQuestion 経由）
**対象**: SPEC-LGX-006 §3 REQ.02 / REQ.10、LGX-COMPAT-001 §4 #4 / §7

## 1. 文脈（Context）

- 背景: GAP-LGX-120。SPEC-LGX-006 REQ.02（旧版）は「個別ノード ID 指定オプションも提供」、REQ.10 は `embed --all --force` に言及していたが、凍結契約 LGX-COMPAT-001 §4 #4 は `embed [--all]` のみを記載しており、フラグの存在が契約上未定義だった。
- 制約: v3 実バイナリの embed は `--all` のみ（`traceability-engine.v3/crates/te-cli/src/main.rs:91-94`。`--json` はグローバルフラグ）。凍結契約への変更はハードルール 7 の対象。
- 関連: SPEC-LGX-006 REQ.02/REQ.10、MCP-INV-1。

## 2. 検討した選択肢（Options）

### 選択肢 A: 加算的拡張として承認（採用）

- 概要: `--node <ID>`（複数可・`--all` 排他）/ `--force` を凍結契約 §4 #4 に追加。既存呼出 `embed [--all]` は挙動不変（v3 に当該フラグが存在しないため後方互換は構造的に保証される）。`embed --json` スキーマ `{generated, skipped, failed, errors[]}` も確定。
- 利点: SPEC が既に要求していた機能（個別再生成・強制再生成）の契約化。既存ユーザへの影響ゼロ。
- 欠点: 「凍結契約は無変更」の原則に例外（加算のみ可）を作る。
- 影響範囲: LGX-COMPAT-001 v1.1.0、SPEC-LGX-006 0.6.0。

### 選択肢 B: 却下して --all のみ維持

- 概要: SPEC-LGX-006 から個別指定・強制再生成の要求を削除。
- 利点: 凍結契約に一切触れない。
- 欠点: モデル差替時の部分検証や単一ノード修正後の再 embed に全件再生成（コスト大）を強いる。SPEC の既存意図を後退させる。

## 3. 判断（Decision）

選択肢 A を採用する（人間裁定 2026-06-10）。

理由:

- v3 にフラグが存在しないため、追加は既存入力空間の挙動を一切変えない純粋な加算であり、互換契約の趣旨（既存呼出の保護）を損なわない。
- 同時に LGX-COMPAT-001 §7 へ「加算的拡張の規律」（SPEC 改訂 + 人間承認 + ADR 必須、意味変更・削除は禁止）を明文化し、なし崩しの拡張を防ぐ。

## 4. 結果（Consequences）

### 期待される効果
- 個別ノード再生成・強制再生成・機械可読な embed 結果が契約として確定し、下流 DD/TS/TC が着手可能になる。

### 受け入れる代償
- 凍結契約のリビジョン管理が必要になる（v1.0.1 → v1.1.0）。

### 残存リスク
- 将来の「加算」が意味変更を伴う混入 → §7 の規律（人間承認 + ADR 必須）でゲートする。
