# ADR-LGX-010: MCP→CLI 子プロセスタイムアウトの導入（既定 30 秒・環境変数可変・0 で無効）

**ステータス**: accepted
**起票日**: 2026-06-10
**承認日**: 2026-06-10
**承認者**: 開発者（人間裁定、AskUserQuestion 経由 2026-06-10）
**対象**: SPEC-LGX-009 §3 REQ.16（新設）、MCP-INV-2、NFR-LGX-001.PERF.03

## 1. 文脈（Context）

- 背景: GAP-LGX-169。Rust CLI 子プロセスがハング（無限ループ・I/O ブロック・デッドロック）した場合、v3 MCP 層にはタイムアウト機構がなく、Agent への応答が永久に返らない（Agent 側のハング）。
- 制約: legixy は v3 と実行時引数互換（LGX-COMPAT-001）。MCP 層は thin forwarder（REQ.09「引数転送ロジックの拡張のみ」）という設計方針がある。

## 2. 検討した選択肢（Options）

### 選択肢 A: タイムアウトなしを明文化（v3 のまま）

- 利点: v3 完全互換・thin forwarder 維持。欠点: 病的ハングで Agent が永久に待つ。

### 選択肢 B: タイムアウト導入（採用・人間裁定）

- 概要: 既定 30 秒、`LGX_MCP_TIMEOUT_SEC` で変更可、`0` で無効（= v3 互換動作へ復帰可能）。SIGTERM → 猶予 5 秒 → SIGKILL、必ず回収（ゾンビ防止）。応答は `isError: true`・`Rust CLI failed (timeout after {N}s):`。部分出力は転送しない（全か無か = MCP-INV-2 保全）。
- 利点: Agent の無限ハングを構造的に排除。正当入力（PERF.03 < 300 ms 予算に対し 100 倍マージン）の挙動は不変。
- 欠点: 【v3 差分】の新挙動。thin forwarder の責務をわずかに超える（プロセスライフサイクル管理）。

## 3. 判断（Decision）

選択肢 B を採用する（人間裁定 2026-06-10。AI 推奨は A だったが、Agent 側可用性を優先する裁定）。

理由:

- 変化するのは病的なハング時のみで、凍結契約（引数体系）と正当入力空間の挙動は不変 — 「凍結契約と正当入力の挙動は不変」の判定基準を満たす。
- `0` 指定で v3 同等動作に戻せる脱出ハッチを設けることで、互換リスクを運用側で制御可能にする。

## 4. 結果（Consequences）

### 期待される効果
- CLI ハング時も Agent は有限時間でエラーを受け取り、リトライ・診断に移れる。ゾンビプロセスが残らない。

### 受け入れる代償
- 30 秒を超える正当処理（将来の超大規模グラフ等）が打ち切られるリスク → 環境変数で引き上げ可能、診断メッセージで案内。

### 残存リスク
- SIGTERM を無視する子プロセス → SIGKILL フォールバックで回収を保証。タイムアウト値の妥当性は実測後に NFR 側で再評価。
