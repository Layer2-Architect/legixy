# ADR-LGX-014: v3↔SPEC 乖離の統治方針 — SPEC 準拠を原則とし乖離は個別 ADR で格上げ

**ステータス**: accepted
**起票日**: 2026-06-13
**承認日**: 2026-06-13（人間裁定）
**対象**: docs/specs-supplement/ SUPP-LGX-001〜010 の [要決定] 中、v3 実装観測挙動と SPEC 字義が乖離する項目群（TRIAGE-2026-06-13 §4）

## 1. 文脈（Context）

- legixy は traceability-engine v0.4.0-alpha4（v3）を OSS リブランドしたもので、実バイナリとの**実行時引数互換**が必須制約（LGX-COMPAT-001）。
- [要決定] トリアージ（`docs/specs-supplement/TRIAGE-2026-06-13_要決定トリアージ.md`）で、SPEC を字義どおり実装すると v3 の観測挙動が変わる項目が多数特定された（compile_context 返却セクション構成、investigate の drift 配線・max-depth、check の DriftFinding kind、migrate の入力受理範囲、drift の exit コード非対称 等）。
- これらを 1 件ずつ「v3 準拠か SPEC 準拠か」で裁定する前に、**統治原則**を 1 つ定めれば残りの個別裁定が従属的に決まる。原則が無いと項目ごとに方針が揺れ、品質偏向防止（00-philosophy §1）に反する。
- 関連: ハードルール 1（SPEC 変更は人間承認）、ハードルール 7（境界 API は DD で凍結、凍結後変更は次版 SPEC 改訂）、ADR-LGX-002（embed --node/--force 加算的拡張の先行例）。

## 2. 検討した選択肢（Options）

### 選択肢 A: SPEC 準拠を原則とし、v3 観測挙動を変える乖離は個別 ADR + 人間承認の加算的拡張として格上げ（採用）

- 概要: SPEC 字義を既定の正とする。v3 の観測挙動を変える実装（例: investigate の suspicious_nodes を実際に出力する）は、それぞれ個別 ADR を起票し人間承認を得て採用する。挙動を変えない範囲では v3 実装を参照実装として流用する。
- 利点: DevProc の加算的拡張規律（既存挙動の後方互換を保ちつつ SPEC 主導で機能追加）に整合。判断の単一情報源が SPEC に保たれる。各乖離の採用根拠が ADR として追跡可能。
- 欠点: v3 観測挙動を変える項目ごとに ADR 起票コストが発生する。

### 選択肢 B: v3 観測挙動を既定維持し、SPEC 側を【v3 差分】注記で追従させる

- 概要: 実装最小・互換最優先。v3 の実測挙動を保ち、SPEC 記述と食い違う箇所は SPEC に【v3 差分】を注記して追従させる。
- 欠点: SPEC が「あるべき仕様」ではなく「現状の追認」に劣化し、前段ループで確定した SPEC の規範性が崩れる。SPEC↔実装の主従が逆転し、品質偏向防止の入力品質（00-philosophy §2.1）を毀損する。
- 不採用。

## 3. 判断（Decision）

選択肢 A を採用する（人間裁定 2026-06-13）。

理由:

- SPEC は前段ループ（QSET→SPP→FCR ACCEPTED）と SPEC レベル TDD（TP/GAP）で既に規範として確定済み。実装はそれに従う（ハードルール 6 の精神）。
- v3 互換が必要なのは**実行時引数契約**（LGX-COMPAT-001 が規定する境界）であって、内部挙動・出力意味論の全てではない。引数契約の凍結を守りつつ、観測挙動の改善（SPEC 準拠化）は加算的拡張として許容される。
- 各乖離採用を個別 ADR にすることで、「なぜ v3 と異なる挙動にしたか」が成果物として残り、多重独立検証経路（00-philosophy §2.3）に資する。

## 4. 結果（Consequences）

### 期待される効果
- TRIAGE §4 の RBA 阻害挙動裁定 16 件の**デフォルトが SPEC 寄り**に定まる（investigate max-depth 機能化・drift 配線、DriftFinding kind 新設、model_version 照合の適用 等）。残るは「要熟慮」4 件（compile_context 返却セクション数、CheckCategory 完全性、observation 状態モデル、drift exit 非対称）の個別議論のみ。
- v3 観測挙動を変える各採用は個別 ADR（ADR-LGX-015 以降）で記録し、必要なら SPEC 改訂（人間承認）+ LGX-COMPAT-001 加算的拡張を連動させる。

### 受け入れる代償
- 乖離項目ごとの ADR 起票コスト。ただし TRIAGE で対象が ~16 件に絞られており管理可能。

### 残存リスク
- 「引数契約の凍結（v3 互換）」と「観測挙動の SPEC 準拠化」の境界が項目によって微妙なケース（例: drift exit コードの非対称）は、加算的拡張がユーザ観測挙動を破壊しないかを ADR で個別精査する必要がある。
