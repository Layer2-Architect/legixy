# SEQD-LGX-011: 閾値キャリブレーション のクラス間メッセージング

**親 RBD**: RBD-LGX-011
**親 SEQA**: SEQA-LGX-011 / **親 UC**: UC-LGX-011
**レイヤ**: 具体側（クラス図レベル、言語非依存）

> **記述規律**: RBD-LGX-011 で識別したクラスをレーンとして、操作呼び出しの時系列を描く。**操作呼び出しは操作名（人間の言語）**。関数名・引数具体型・戻り型・言語固有同期機構は書かない（DD で確定）。本 SEQD は **Behavior Allocation**（どのクラスがどの操作を担うか）を確定する。
>
> **ハードルール 10**: 命名規則に従う関数呼び出し・言語固有のジェネリック型・並行修飾子・モジュール識別子が混入したら違反。`scripts/trace-check.sh` [5/5] が検出する。本ファイルは禁止トークンを literal で引用しない（記述的に書く）。

## 4. 並行性（概念レベル）

`calibrate` は読み取り専用の分析処理であり、ドメインレベルの並行性はない。埋め込みロード → 全ペア算出 → ヒストグラム生成 → 推奨閾値算出（フラグ指定時）→ 結果整形 の各処理はキャリブレーション統括処理の協調下で逐次進む。SCORE-INV-1（決定性保証: 同一埋め込みベクトル集合 → 同一ヒストグラム）がこの逐次性と整合する。具体的な並行機構は DD で扱う。

## 5. 失敗伝搬

- 各操作の戻り値は「結果」概念（成功 / 失敗 + 理由）で表現。具体的なエラー型は DD で確定。
- バケット数 0 の入力契約違反はキャリブレーション統括処理が終了状態種別（失敗）として出力窓口へ伝搬する。
- 全ペア算出失敗は算出処理からの失敗結果として統括処理へ伝搬し、統括処理が出力窓口へエラーを渡す。
- 空ストア（早期終了）および推奨閾値算出不能（ペア数 0）は情報通知として標準エラー出力へ渡し、終了状態種別は成功のまま維持する（出力先分離の観測要件準拠）。

## 6. Behavior Allocation（操作のクラス帰属）

各操作は一つのクラスに帰属する（複数クラスへの分散なし）。Boundary=境界操作のみ / Control=複数 Entity の協調 / Entity=自身のデータ操作。

| 操作 | 帰属クラス | 役割 | 妥当性 |
|---|---|---|---|
| キャリブレーション要求を受け付ける / オプションを検証する | キャリブレーションコマンド受付窓口 | Boundary（アクター境界） | ✓ 境界操作のみ |
| キャリブレーションを統括する / 早期終了を判定する | キャリブレーション統括処理 | Control（協調） | ✓ |
| 埋め込みベクトルをロードする / 空ストアを検出する | 埋め込みロード処理 | Control | ✓ |
| 全件を読み出す / ストアが空かを確認する | 埋め込みストア境界 | Boundary（外部ストア境界） | ✓ |
| 件数を取り出す | 埋め込みベクトル集合 | Entity（自身のデータ） | ✓ |
| 全ペア類似度を算出する / 非有限スコアのペアを除外する | 全ペア類似度算出処理 | Control | ✓ |
| ペア数を取り出す / スコア一覧を取り出す | 全ペア類似度集合 | Entity（自身のデータ） | ✓ |
| ヒストグラムを生成する / 統計サマリを生成する | ヒストグラム生成処理 | Control | ✓ |
| バケット一覧を取り出す | ヒストグラム | Entity（自身のデータ） | ✓ |
| 統計値を取り出す | 統計サマリ | Entity（自身のデータ） | ✓ |
| 推奨閾値を算出する / ペア数ゼロを検出する | 推奨閾値算出処理 | Control | ✓ |
| 推奨値一覧を取り出す | 推奨閾値 | Entity（自身のデータ） | ✓ |
| キャリブレーション結果を生成する / 推奨閾値を結果に加える | 結果整形処理 | Control | ✓ |
| 閾値設定を読み出す | 設定ファイル境界 | Boundary（外部ファイル境界） | ✓ |
| 閾値一覧を取り出す | 現閾値設定 | Entity（自身のデータ） | ✓ |
| 結果を取り出す | キャリブレーション結果 | Entity（自身のデータ） | ✓ |
| 結果を標準出力へ渡す / 情報通知を標準エラー出力へ渡す / エラーを標準エラー出力へ渡す | キャリブレーション結果出力窓口 | Boundary（出力境界） | ✓ |

割り当てに迷う操作なし。各操作が UC ステップ / SEQA メッセージ / RBD 操作識別と対応（余剰操作なし）。
