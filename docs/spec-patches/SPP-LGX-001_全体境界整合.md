## 承認手順

1. 開発者が各差分を確認（差分 7 / 差分 8 は QSET 外の編集整理のため個別判断可）

3. 全差分が妥当であれば **承認**（本 SPP のヘッダの承認状態を「承認済」に更新）

5. 反映後、フロントエンド検証を実行して FCR-LGX-001 を発行

- 追加質問を載せた新規 QSET を発行して次の反復に進む。QSET の採番は**次の空き連番**を払い出し、本文 metadata の「親 SPEC」+「反復回数」で対応付ける（QSET-LGX-010 は SPEC-LGX-010 の前段ループ用に予約済みのため使用しない）
