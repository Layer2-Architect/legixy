## 4. Role Fitness Check（§5.2）

### Boundary

- **グラフ定義境界**: 読み込み・存在確認の境界操作のみ保持。グラフ解析や内容解釈は Control（C0）が担う。Boundary overreach なし。
- **ドキュメントファイル境界**: ファイル読み込み・存在確認の境界操作のみ保持。見出し抽出は Control（C1）が担う。Boundary overreach なし。
- Finding: なし / Severity: なし / Required action: なし

### Control

- **サブノード抽出統括処理**: 起動・取り込み・マージ依頼・loop 協調のみ。データ保持なし（属性なし）。Service blob 化なし（他 Control の内部ロジックに立ち入らない）。
- **見出し抽出処理**: ファイル読み込み依頼・正規化依頼・階層パス構築・ID 生成依頼のみ。正規化や ID 計算を自身で行わない。Control leakage なし。
- **見出しテキスト正規化処理**: 正規化変換ロジックのみ。抽出・生成越権なし（属性なし）。
- **サブノード ID 生成処理**: ハッシュ構築・ID 確定のみ。マージ・グラフ更新は行わない。Service blob 化なし。
- **サブノードマージ処理**: 照合・衝突解決・縮退・グラフ反映のみ。見出し抽出・ID 生成は行わない。越権なし。
- Finding: なし / Severity: なし / Required action: なし

### Entity

- **ドキュメントノード**: 識別子・ファイルパス・ノード種別を保持し、自身の親識別子を返す操作のみ。他 Entity の操作を代行しない。Entity anemia / overreach なし。
- **見出し階層パス**: 上位・対象テキスト・深さを保持し、自身の階層パス文字列を構築する操作のみ。Entity anemia / overreach なし。
- **自動生成サブノード**: 識別子・見出し階層パスへの参照・親 ID・種別を保持し、自身から派生する親子エッジを生成する操作のみ。Entityanemia / overreach なし。
- **明示サブノード**: 識別子・名称・親 ID・種別を保持する（操作なし）。照合は C4 が担う。保持専用 Entity として妥当（Entity anemia ではなく設計的に純粋保持 Entity）。
- **有向グラフ**: ノード・サブノード・エッジのコレクションを保持し、追加・参照操作を提供。他 Entity の内部を操作しない。Entity anemia / overreach なし。
- Finding: なし / Severity: なし / Required action: なし

**注（SEQA vs SEQD のサブノード ID 生成処理呼び出し者）**: SEQA §2 の基本フロー図では `C1->>C3: サブノード ID 生成を依頼する` と表現されているが、SEQD §1 では `C0->>C3: サブノード ID を確定する` となっており、呼び出し者が C1 → C0 にシフトしている。この差異を詳細に検査する。

- RBA-LGX-003 §2 の関係記述「サブノード ID 生成処理 は 見出し階層パス と ドキュメントノード（親 ID）から 自動生成サブノード の ID を確定する」では呼び出し者を明示していない。
- RBD-LGX-003 §2 クラス間関係では「サブノード抽出統括処理 → 見出し抽出処理 / サブノード ID 生成処理 / サブノードマージ処理: 依存（統括処理が各処理を協調利用する）」とあり、C0 が C3 を呼ぶ構造が RBD で明示されている。
- SEQD の `C0→C3` は RBD §2 と一致する。SEQA の `C1→C3` はドメインレベルの「見出し抽出後に ID 生成を依頼する」という概念的フローを表現したもの（依頼者の帰属先がドメインレベルでは曖昧のまま）。
- 具体的な帰属先を RBD 段階で C0（統括処理）に確定したことは、Object Discovery ではなく Behavior Allocation の正当な具体化であり、SEQA が記述する「責務の担い手」は変わっていない（サブノード ID 生成処理が ID を確定するという責務は保存済み）。
- **判定**: shifted / mutated なし。SEQA の C1→C3 表現と SEQD の C0→C3 表現は、呼び出し者の帰属先を抽象→具体で具体化したものであり、サブノード ID 生成処理の責務（AR-006）は CR-006 として保存されている。ambiguous 解消済み。
