# GAP-LGX-001: UC 網羅宣言（001〜011）と SPEC-LGX-010 のスコープ拡張（012/013）の不一致

**親 TP**: TP-LGX-001
**観点出典**: TP-LGX-001 §2.1 観点 4（カテゴリ網羅性 — umbrella 網羅宣言と下位 SPEC 対応 UC 集合の件数・範囲一致）
**ステータス**: closed (2026-06-10)
**起票日**: 2026-06-08
**severity**: minor
**敵対的精査メモ（2026-06-09）**: GENUINE と確認。事実関係を検証済み（SPEC-LGX-010 §1.3 line 35 が「REQ.01/REQ.02 を 001〜013 へ再改訂する必要がある」と明記、UC-LGX-012/013 はファイル・graph.toml ノードとも不在、umbrella は 001〜011 のまま）。現時点で umbrella↔下位 SPEC の partition 宣言が実際に open で不一致しており、UC フェーズで孤児 UC を誘発する追跡上の実害がある。「将来 SPP 反復で同期予告済み」は子 SPEC の片務予告であって umbrella 側に予約注記が無いため、読者には見落としと区別不能。ALREADY_ANSWERED/OUT_OF_SCOPE/DUPLICATE いずれにも該当せず。severity は minor（修正は予約注記の追記 + UC フェーズでの同期のみ）。

## 1. 観点

umbrella の網羅宣言（REQ.01/REQ.02 の「UC-LGX-001〜011」）と、下位 SPEC が宣言する対応 UC 集合の範囲が一致しているか。

## 2. 現状の SPEC / UC

SPEC-LGX-001 §3（REQ.01 検証方法・REQ.02 根拠/検証方法）および表ヘッダ「対応 UC: UC-LGX-001〜011（全体）」は、対応 UC 範囲を **001〜011** と宣言している。

一方、SPEC-LGX-010 §1.3 は「本 SPEC の 4 コマンドは UC-LGX-010 / 011 / **012（snapshot）** / **013（drift）** に写像する。UC-012/013 は本 SPEC 受理後の UC フェーズで新規生成する」と宣言し、さらに「UC-LGX-012/013 の新規生成に伴い、**SPEC-LGX-001 REQ.01/REQ.02 の網羅宣言『UC-LGX-001〜011』は『001〜013』へ再改訂する必要がある**」と明記している。

しかし現時点で UC-LGX-012/013 はファイルとしても graph.toml ノードとしても存在せず（確認済）、umbrella の網羅宣言は 001〜011 のまま据え置かれている。すなわち umbrella と下位 SPEC-LGX-010 の間に **partition 宣言の不一致が open のまま** 存在する。

## 3. 期待される情報

SPEC または UC に追加されるべき記述（適用は人間承認後・UC フェーズ着手時）:

- SPEC-LGX-001 REQ.01/REQ.02 および表ヘッダの「UC-LGX-001〜011」を、UC-012/013 生成と同期して「001〜013」へ再改訂する（SPEC-LGX-010 §1.3 が予告する SPP-LGX-001 次反復として）。
- それまでの期間、umbrella 側に「UC-012/013 は SPEC-LGX-010 受理に伴い予約済・未生成。再改訂は UC フェーズ着手時」という暫定注記を REQ.02 直近に置き、宣言の不一致が見落としでなく既知の予約状態であることを読者に明示する。
- 再改訂の責任者・トリガ（どの成果物の確定でいつ反映するか）を観点 17（GAP-LGX-004）と整合する形で規定する。

## 4. 影響範囲

この GAP がクローズされないと以下に影響:

- UC フェーズ: UC-LGX-012/013 を生成しても umbrella の網羅宣言が追従せず、REQ.02 検証方法「各 UC が上記いずれかのカテゴリに対応すること」の機械的・人手検証が宣言件数と食い違う。
- 下流の TP[UC] / RBA 以降: SPEC-LGX-010 を親とする UC-012/013 のトレーサビリティが umbrella 網羅宣言から漏れ、孤児 UC として検出されうる。
- 他の GAP との依存: GAP-LGX-004（partition 統治手順の不在）と同根。統治手順が確立すれば本 GAP の再発防止になる。

## 5. 解決経緯（クローズ時に記入）

SPEC-LGX-001 v0.7.0（人間承認 2026-06-10）: ヘッダ表「対応 UC」に UC-012/013 予約注記を追加、REQ.02 末尾に予約注記を追記（本文「001〜011」は既知の予約状態として据え置き。再改訂は UC フェーズ着手時に §7 変更ポリシーに従い実施）。

## 6. 関連 ADR

- partition 拡張の同期タイミングを「UC フェーズ着手時の SPP 次反復で処理」と決定する判断は ADR 化候補（前段スキップ記録ではなく partition 改訂手順の決定）。必要に応じ ADR-LGX-NNN を起票。
