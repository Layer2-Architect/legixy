# GAP-LGX-283: calibrate 次元不一致スキップ（集約 Warning）の失敗パスが UC に未記述

**親 TP**: TP-LGX-021
**観点**: §2.3 EF1 次元不一致スキップの欠落
**ステータス**: closed (2026-06-13)
**起票日**: 2026-06-13
**分類（暫定）**: WEAK

## 1. 観点

SPEC-LGX-010 REQ.05 は次元不一致ペアをスキップして処理継続し、集約 Warning 1 件を stderr に出力することを規定する（【v3 差分】）。UC-LGX-011 のフロー記述にこの部分失敗継続パスが現れておらず、事後条件「標準出力にヒストグラム + 閾値が出力される」が達成される際の stderr 出力が観察可能化されていない。

## 2. 現状の UC / SPEC

**UC-LGX-011 の記述:**
- Step3「全ペア類似度を算出する（O(N²)）」— スキップ挙動への言及なし
- 代替フロー 3a「全ペア算出失敗時: anyhow エラーコンテキスト付きで exit 1」— 「全失敗」のみ。「部分スキップ＋継続」パスなし
- 事後条件「engine.db は不変（読取のみ）」は記述されているが集約 Warning の出力は未記載

**SPEC-LGX-010 REQ.05 の規定:**
- 全ペア類似度から次元不一致ペアをスキップし、スキップ発生時は集約 Warning 1 件を stderr に出力する【v3 差分】
- 同様に非有限スコア（NaN/±Inf）も skip + 集約 Warning（REQ.09）
- 参照: TP-LGX-010 E10（集約 Warning 1 件 stderr）が GREEN 確立済み

これは SPEC-LGX-010 REQ.05 が既に回答済みの挙動である。

## 3. 推奨対応（人間裁定）

**(A) UC へ追記案**: 代替フローに以下を追加する:
- `3b. 次元不一致ペアが存在する場合: 当該ペアをスキップし集約 Warning 1 件（stderr）を出力して処理を継続する（exit 0）`

**(B) drop（委譲容認）案**: 次元不一致スキップは SPEC-LGX-010 REQ.05 および TP-LGX-010 E10 が所有するエンジン挙動。UC は主要な制御フロー（成功 / 空ストア / バリデーション失敗 / 全体失敗）を記述すれば足りる。部分スキップ継続は SPEC 委譲として UC への明示列挙は不要と判断する。

## 4. 影響範囲

- UC-LGX-011 §代替フロー（追記案 A の場合）
- GAP-LGX-284（非有限スコアの失敗パス欠落）: 同性質の WEAK であり本 GAP の裁定方針を共用できる
- GAP-LGX-281（代替フロー網羅性）: 次元不一致スキップはその一部として記述されており、本 GAP close により GAP-281 も部分的に解消される

## 5. 解消（2026-06-13）

敵対的精査裁定: **方針B（委譲容認）**。本観点は親 SPEC（対応 TP[SPEC] green 確定済）が回答しており、UC フローへの明示列挙は任意。GREEN-by-delegation として close。詳細: docs/spec-change-proposals/2026-06-13_uc-gap-adversarial-review.md §F。
