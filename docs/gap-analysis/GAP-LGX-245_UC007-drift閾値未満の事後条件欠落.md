# GAP-LGX-245: UC-LGX-007 drift の閾値出所と閾値未満時の事後条件が未定義

**親 TP**: TP-LGX-017
**観点**: §2.6 R4
**ステータス**: closed (2026-06-13)
**起票日**: 2026-06-13
**分類（暫定）**: WEAK

## 1. 観点

UC-LGX-007 の drift フロー Step4「ドリフトが閾値以上の場合、リンクされた成果物のレビューを推奨する」において、①閾値の出所（設定ファイル参照か、どの設定キーか）および②閾値未満の場合の明示的な事後条件（「推奨なし」の状態が観察可能か）が UC フロー記述から読み取れない。

## 2. 現状の UC / SPEC

**UC-LGX-007 drift フロー Step4（現行）:**
> 4. ドリフトが閾値以上の場合、リンクされた成果物のレビューを推奨する

閾値の出所と閾値未満の case が未記述。

**SPEC-LGX-010.REQ.03 の規定（既存）:**
> drift 値の定義: `drift = 1.0 − cosine 類似度`。値域 [0.0, 2.0]

SPEC-LGX-010.REQ.03 は drift 値の定義・JSON スキーマ・ベースライン選択規則を規定するが、閾値の数値既定は NFR-LGX-001 / `.legixy.toml` に委ねられている（SPEC-LGX-010 §1 「含まない: 閾値の数値既定（→ NFR-LGX-001 / `.legixy.toml`）」）。

UC フロー記述への反映が欠落しているが、SPEC が閾値管理の所在を明示している。

## 3. 推奨対応（人間裁定）

**(A) UC へ追記案:**
Step4 を以下のように展開する:
> 4. drift 値と設定ファイルの閾値（`drift_threshold`）を比較する:
>    - 4a. 閾値以上の場合: リンクされた成果物のレビューを推奨する（drift 値とともに出力）
>    - 4b. 閾値未満の場合: drift 値のみ出力し、レビュー推奨は行わない

**(B) drop（委譲容認）案:**
閾値の数値・設定キーは NFR-LGX-001 / `.legixy.toml` が所有しており、UC レベルでの具体的閾値記述は範囲外。「閾値未満の case は推奨なし」は「閾値以上の場合に推奨する」の論理的裏面として自明。UC フローへの追記は不要と裁定する。

## 4. 影響範囲

- UC-LGX-007 drift フロー Step4 の記述明確化（閾値出所と閾値未満の case 追加）
- 後続 RBA での閾値比較ロジックの設計根拠
- WEAK 確定の場合は UC フロー変更不要

## 5. 解消（2026-06-13）

敵対的精査裁定: **方針B（委譲容認）**。本観点は親 SPEC（対応 TP[SPEC] green 確定済）が回答しており、UC フローへの明示列挙は任意。GREEN-by-delegation として close。詳細: docs/spec-change-proposals/2026-06-13_uc-gap-adversarial-review.md §F。
