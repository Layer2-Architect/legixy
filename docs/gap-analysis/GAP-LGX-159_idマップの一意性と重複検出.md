# GAP-LGX-159: id-map の新旧 ID 一意性検証・衝突検出が未定義

**親 TP**: TP-LGX-008
**観点出典**: TP-LGX-008 §2.7 観点 I-5
**ステータス**: closed (2026-06-10)
**起票日**: 2026-06-08

## 1. 観点

REQ.11 は `.legixy/migration-id-map.toml` を生成すると述べるが、**新旧 ID の一意性検証（複数の旧 ID が同一の新 ID にマップされる衝突、または同一旧 ID が複数の新 ID に対応する曖昧性）の検出と扱い** が定義されていない。マッピングが多対一/一対多になると ID 書き換え（from/to/parent）が破綻する。

## 2. 現状の SPEC / UC

SPEC-LGX-008 §3 REQ.11 で **「旧 ID と新 ID の対応表を生成」「参照を新 ID に書き換え」** に触れ、マッピング不可の ID は Warning（GAP-LGX-146 で別途）とするが、**マッピングが衝突・重複する場合の検証と扱い** は未定義。新 ID は SHA-256 ベースで一意のはずだが、旧 ID 側の重複や正規化差異による衝突の検出責任が不明。

## 3. 期待される情報

SPEC または UC に追加されるべき記述:

- id-map における新旧 ID の一意性制約（一対一を保証するか、多対一/一対多を許すか）
- 衝突検出時の挙動（Error / Warning / 人間確認）
- SUBNODE-INV-3（ID 一意性）との整合 — 生成された新 ID が graph 全体で一意であることの保証範囲

## 4. 影響範囲

この GAP がクローズされないと以下に影響:

- 下流 TS / TC: id-map 一意性・衝突テストの期待値が書けない
- SUBNODE-INV-3（ID 一意性）侵害のリスク
- 他の GAP との依存: GAP-LGX-146（マッピング不可 ID）

## 5. 解決経緯（クローズ時に記入）

SPEC-LGX-008 v0.6.0（人間承認 2026-06-10）: REQ.11 に id-map の旧→新 ID 全単射保証（旧 ID 重複=曖昧性 Error・新 ID 衝突=多対一 Error・graph 全体一意性 SUBNODE-INV-3 違反 Error、--dry-run でも検証）を確定。§4 SUBNODE-INV-3 行更新。
