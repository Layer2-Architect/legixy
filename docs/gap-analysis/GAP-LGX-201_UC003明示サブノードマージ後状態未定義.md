# GAP-LGX-201: UC-LGX-003 Step3 明示サブノードマージ後の事後条件未定義

**親 TP**: TP-LGX-013
**観点**: §2.1 BF3
**ステータス**: closed (2026-06-13)
**起票日**: 2026-06-13
**分類（暫定）**: WEAK

## 1. 観点

UC-LGX-003 Step3「graph.toml で明示定義されたサブノード（`#s:` 接頭辞）もマージする」は手順として記述されているが、自動生成済みサブノードとのマージ後の状態（グラフ上の統一性・明示サブノードへの ParentChild エッジ生成可否・明示サブノードの path が親と一致するかどうか）が UC の事後条件に明示されていない。

## 2. 現状の UC / SPEC

- UC-LGX-003 Step3: 「graph.toml で明示定義されたサブノード（`#s:` 接頭辞）もマージする」——手順は記述されているが、マージ後のグラフ状態が事後条件に反映されていない。
- UC-LGX-003 事後条件は自動生成サブノードの性質（SUBNODE-INV-1/2/5/6）のみを列挙し、明示サブノードがマージされた後の統合状態を観察可能に記述していない。
- SPEC-LGX-002.REQ.05 は自動生成と明示の 2 方式を共存規定。REQ.03 はサブノードの path フィールドを必須化。SUBNODE-INV-1（親存在）・SUBNODE-INV-2（path 整合性）は明示サブノードにも適用される。

## 3. 推奨対応（人間裁定）

(A) **UC へ追記案**: UC-LGX-003 の事後条件に「自動生成サブノードと明示サブノードが同一グラフに統合され、明示サブノードも SUBNODE-INV-1（親存在）・SUBNODE-INV-2（path 整合性）を満たすこと」を追記する。

(B) **drop（委譲容認）案**: SPEC-LGX-002.REQ.05 が 2 方式の共存を規定し、REQ.03/SUBNODE-INV-1/SUBNODE-INV-2 が明示サブノードにも適用されることは TP-LGX-002 の GREEN 観点群で確立済みであるため、UC フロー記述への重複明示は不要として drop する。

## 4. 影響範囲

- UC-LGX-003 §事後条件
- 下流成果物（RBA/SEQA/DD/TS/TC）で明示サブノードの合流点を設計する際、マージ後状態の明確化が前提となる可能性がある。影響度は低（SPEC-LGX-002 が既に答えている）。

## 5. 解消（2026-06-13）
