# GAP-LGX-231: UC-LGX-006 が Step4「返却する」と事後条件「標準出力に表示」の橋渡しを欠く

**親 TP**: TP-LGX-016
**観点**: 2.1 BF1（各ステップの事後条件が後続ステップの前提を満たすか）
**ステータス**: closed (2026-06-13)
**起票日**: 2026-06-13
**分類（暫定）**: WEAK（SPEC-005 委譲で解決可。「返却する」=stdout 出力と読める余地あり）

## 1. 観点

UC レベル観点「各ステップの事後条件が後続ステップの前提を満たすか」を UC-LGX-006 の基本フロー Step4 と事後条件の連鎖にぶつけた。

## 2. 現状の UC / SPEC

- UC-LGX-006 基本フロー Step4 は「結果を以下の形式で返却する: visited / depth_map」と記述し、フローはここで終端する。
- 事後条件「走査結果が標準出力に表示される」はフロー外の事後条件節に記述されており、Step4 の「返却する」と事後条件の「標準出力に表示される」の間を繋ぐ「出力・表示」ステップが基本フロー上に現れていない。
- UC-LGX-001 では Step5「システムが検証結果を標準出力に表示する」/ Step6「exit コードで終了する」という独立ステップを設けており、フローと事後条件の連鎖が明示的。
- SPEC-LGX-005.REQ.09 で走査結果の出力要件は規定済だが、それは SPEC レベルの規定であり、UC フロー記述として観察可能なステップが欠落している状態に変わりはない。

## 3. 推奨対応（人間裁定）

以下のいずれか:

- **(A) UC へ追記**: 基本フロー Step5「システムが走査結果（visited / depth_map）を標準出力に表示する」Step6「exit 0 で終了する」を追加し、事後条件との連鎖を明示する。
- **(B) drop（委譲容認）**: Step4 の「返却する」を「標準出力へ出力する」と等価な表現と認め、SPEC-LGX-005.REQ.09 への委譲で UC フロー記述として十分と認める。→ UC は変更しない。

WEAK 候補（「返却する」の解釈粒度に依存）。
