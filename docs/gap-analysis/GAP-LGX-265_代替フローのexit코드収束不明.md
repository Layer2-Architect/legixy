# GAP-LGX-265: UC 代替フロー 2a/2b の exit コード収束が不明

**親 TP**: TP-LGX-019
**観点**: §2.2 AF3
**ステータス**: closed (2026-06-13)
**起票日**: 2026-06-13
**分類（暫定）**: WEAK

## 1. 観点

UC-LGX-009 の代替フロー 2a（init 既存ファイル）と 2b（migrate 対象未発見）はともに「ERROR を報告する」で終了するが、LGX-COMPAT-001 §3 の終了コード規約（exit 1 = 実行時エラー / exit 2 = 使用法誤り）のどちらへ収束するかが UC に観察可能でない。

## 2. 現状の UC / SPEC

UC-LGX-009 の代替フロー:
- **2a.** init で `.legixy.toml` が既に存在する場合、ERROR を報告する
- **2b.** migrate で v0.1.0 プロジェクトが見つからない場合、ERROR を報告する

LGX-COMPAT-001 §3 の凍結終了コード規約:
- `0` = 成功
- `1` = 実行時エラー（例: ファイル読み込み失敗・変換失敗）
- `2` = 使用法誤り（例: 必須引数なし・不正フラグ）

2a（既存ファイルあり）は、引数は正しいが実行が許可されない状態 → exit 1（実行時エラー）と解釈できるが、「使用法として init を実行できる状態でない」とも解釈できる。2b（プロジェクト未発見）も同様に exit 1 か exit 2 か曖昧。

TP-LGX-008 F-4（終了コード規約確立）は全コマンド共通の GREEN 観点として確立しているが、UC のフロー記述レベルで代替フロー終了時の exit コード収束が観察可能でない。

## 3. 推奨対応（人間裁定）

**(A) UC へ追記案:**
代替フローの終了記述を具体化する:
- **2a.** init で legixy 管理生成物が既に存在する場合、ERROR を報告して終了する（exit 1）
- **2b.** migrate で v0.1.0 プロジェクトが見つからない / 読めない場合、ERROR を報告して終了する（exit 1）

**(B) drop（委譲容認）案:**
LGX-COMPAT-001 §3 凍結規約 + TP-LGX-008 F-4 が所有する観点として委譲確定とする。UC フロー記述として「ERROR を報告する」は十分であり、exit コードの具体値は TS/DD レベルで確定。

## 4. 影響範囲

- 下流 TS: init/migrate の各エラーパス終了コード検証テスト
- CI スクリプトが exit コードを判定する場合の期待値

## 5. 解消（2026-06-13）

敵対的精査裁定: **方針B（委譲容認）**。本観点は親 SPEC（対応 TP[SPEC] green 確定済）が回答しており、UC フローへの明示列挙は任意。GREEN-by-delegation として close。詳細: docs/spec-change-proposals/2026-06-13_uc-gap-adversarial-review.md §F。
