# GAP-LGX-205: UC-LGX-003 Step2c 正規化の具体内容が SPEC-LGX-002.REQ.06 委譲を明示していない

**親 TP**: TP-LGX-013
**観点**: §2.6 R1
**ステータス**: closed (2026-06-13)
**起票日**: 2026-06-13
**分類（暫定）**: WEAK

## 1. 観点

UC-LGX-003 基本フロー Step2c「各見出しテキストを正規化する（トリム + 連続空白統合）」は SPEC-LGX-002.REQ.06 が規定する正規化規則のサブセットのみを例示している。REQ.06 が必須とする以下の項目が UC フロー記述に現れていない:

- Markdown 装飾文字の除去（`**`, `__`, `*`, `_`, `` ` ``, `~~`）
- 全角スペース（U+3000）の半角スペースへの変換
- Unicode NFC 正規化

REQ.06 への委譲も明示されていないため、UC の Step2c を読んだだけでは正規化の完全な規則が把握できない。

## 2. 現状の UC / SPEC

- UC-LGX-003 Step2c: 「各見出しテキストを正規化する（トリム + 連続空白統合）」——一部の正規化ルールのみを例示。
- SPEC-LGX-002.REQ.06: 前後空白削除・連続空白統合・Markdown 装飾文字除去・全角スペース正規化・NFC 適用 の 5 規則を明示。また「これらの正規化により、Markdown フォーマッタ（Prettier 等）による微細な文字列変化が ID 不安定性を引き起こさないことを保証する」と目的も明記。
- TP-LGX-002 G-IV-1（GREEN 確立済）: 正規化 5 規則は REQ.06 で確立済み。

## 3. 推奨対応（人間裁定）

(A) **UC へ追記案**: Step2c を「各見出しテキストを SPEC-LGX-002.REQ.06 に従い正規化する（前後空白削除・連続空白統合・Markdown 装飾除去・全角スペース正規化・NFC）」と修正し、委譲先を明示する。

(B) **drop（委譲容認）案**: SPEC-LGX-002.REQ.06 が完全に 5 規則を規定しており、UC フロー記述での例示は委譲元の参照として十分と解釈して drop する。実装時は必ず REQ.06 を参照する旨を notes に残すのみ。

## 4. 影響範囲

- UC-LGX-003 §基本フロー Step2c
- 影響度は中程度。下流成果物（DD/TS/TC）で Step2c を実装する場合に正規化が不完全になるリスクがある。TP-LGX-002 は G-IV-1 で GREEN 確立済みだが、UC フローの Step2c 記述を参照した実装者が REQ.06 全規則を見落とす可能性がある。
- ID 安定性（SUBNODE-INV-5 の実用的担保）に直接関わるため、down-stream への潜在的影響は実用上無視できない。
