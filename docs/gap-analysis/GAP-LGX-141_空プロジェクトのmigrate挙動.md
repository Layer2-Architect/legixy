# GAP-LGX-141: 成果物 0 件の空 v0.1.0 プロジェクトを migrate したときの挙動が未定義

**親 TP**: TP-LGX-008
**観点出典**: TP-LGX-008 §2.1 観点 B-1
**ステータス**: closed (2026-06-10)
**起票日**: 2026-06-08

## 1. 観点

成果物が 0 件（空 engine.db / ノードを含まない matrix.md / 空 `.legixy.toml`）の v0.1.0 プロジェクトを `migrate` したとき、空の `graph.toml` を生成して成功とするのか、入力不正として Error を返すのかが定義されていない。

## 2. 現状の SPEC / UC

SPEC-LGX-008 §3 REQ.03 で **matrix.md の各成果物 ID をノードとして抽出し chain 順でエッジを生成する** 変換ルールに触れているが、**抽出対象が 0 件のときの出力（空 graph.toml で成功か Error か）** は未定義。REQ.07（init）は空ディレクトリからの初期化を扱うが、空 v0.1.0 プロジェクトの migrate とは別経路。

## 3. 期待される情報

SPEC または UC に追加されるべき記述:

- 成果物 0 件の v0.1.0 入力に対し、空（ノード 0・エッジ 0）の graph.toml を生成して正常終了とすることの明示
- もし Error とするなら、その理由と終了コード、ユーザ向けメッセージ
- 「マイグレーション対象が無い」と「マイグレーション対象が壊れている」の区別

## 4. 影響範囲

この GAP がクローズされないと以下に影響:

- UC-LGX-009: 空プロジェクト移行フローの事後条件が確定できない
- 下流 TS / TC: 境界値テスト（空入力）の期待値が書けない
- 他の GAP との依存: GAP-LGX-144（source データ破損検出）と「対象なし vs 破損」の境界で関連

## 5. 解決経緯（クローズ時に記入）

SPEC-LGX-008 v0.7.0（人間裁定 fix・承認 2026-06-10）: REQ.03 に空入力挙動を確定 — 成果物 0 件は空 graph.toml（ノード 0/エッジ 0）生成で exit 0 + stderr Info。「対象なし」（パース成功 + 0 件）と「破損」（REQ.03a）の区別を明記。
