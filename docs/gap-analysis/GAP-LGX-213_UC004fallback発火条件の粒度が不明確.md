# GAP-LGX-213: UC-LGX-004 代替フロー 3a fallback の発火条件・適用粒度がフロー記述に観察可能でない

**親 TP**: TP-LGX-014
**観点**: §2.2 AF2（fallback 発火条件の不明確さ）
**ステータス**: closed (2026-06-13)
**起票日**: 2026-06-13
**分類（暫定）**: GENUINE

## 1. 観点

UC-LGX-004 代替フロー 3a「サブノードが存在しない上流成果物は、ドキュメント全体として返却する（fallback、Phase 2 Block B 後も維持）」において、(1) サブノード有無の判定タイミング（Step 3a でエッジを辿る最中か、事前の一括検査か）と (2) fallback の適用粒度（上流成果物の全てがサブノード不在の場合か、個々の成果物単位でノードごとに fallback が発動するか）がフロー記述から観察できない。

## 2. 現状の UC / SPEC

- **UC-LGX-004 代替フロー 3a**: 「サブノードが存在しない上流成果物は、ドキュメント全体として返却する（fallback）」。「上流成果物」が単数形か集合か、また「ドキュメント全体」が当該ノードを指すのかが不明確。
- **UC-LGX-004 事後条件**: 「サブノードが存在しない場合のみ親全文へ fallback」という記述はノード単位での適用を示唆するが、フロー記述の 3a と対応する詳細ステップが存在しない。
- **SPEC-LGX-003**: 動的 fallback を規定するが、ノード単位か集合単位かの明示はない。
- **問題点**: サブノードを持つノードとサブノードを持たないノードが上流成果物に混在する場合の挙動（混在時は一部サブノード展開 + 一部ドキュメント全文返却）がフロー記述に現れていない。これは RBA / DD レベルの設計判断に影響する。

## 3. 推奨対応（人間裁定）

**(A) UC への追記案**

代替フロー 3a を「上流成果物ごとにサブノード有無を確認し、サブノードが存在しない成果物については個別にドキュメント全体を UpstreamArtifact として返却する（ノード単位の部分 fallback）」に明確化する。また基本フロー Step 3 に「各上流成果物についてサブノード有無を確認する条件分岐」を示す記述を追加する。

**(B) drop（委譲容認）案**

「ドキュメント全体として返却する（fallback）」の粒度はノード単位であることが「上流成果物は」という複数形の文脈から自然に読み取れるとみなし、SPEC-LGX-003 の規定に委譲。UC フロー修正不要と裁定する。

## 4. 影響範囲

- UC-LGX-004 §代替フロー（追記案の場合）・§基本フロー Step 3
- 下流への影響: RBA でサブノード有無の条件分岐がドメインオブジェクトのどのロールに帰属するかの設計に直接影響する。GENUINE 確定の場合は RBA 着手前に解消が望ましい

## 5. 解消（2026-06-13）

敵対的精査裁定: **方針B（委譲容認）**。本観点は親 SPEC（対応 TP[SPEC] green 確定済）が回答しており、UC フローへの明示列挙は任意。GREEN-by-delegation として close。詳細: docs/spec-change-proposals/2026-06-13_uc-gap-adversarial-review.md §F。
