# GAP-LGX-101: 空テキスト・空白のみノードの embedding 生成挙動が未定義

**親 TP**: TP-LGX-006
**観点出典**: TP-LGX-006 §2.1 観点 B-01
**ステータス**: closed (2026-06-10)
**起票日**: 2026-06-08
**重要度**: minor (verification: low-value, 人間判断で drop 可) — 敵対的精査パス 2026-06-09: degenerate edge（全ノードに最低 Document ID 行が存在し真の空本文は稀）。数値安定性の核は GAP-LGX-104 に集約。WEAK_OR_PADDED として維持。

## 1. 観点

本文が空、または空白・改行のみのノード（および content_range が空のサブノード）に対して `embed` が embedding を生成する際、トークン列が空またはほぼ空になる。mean pooling は空トークン列で 0 除算しうる。この場合に生成されるベクトルの定義（ゼロベクトルか、生成スキップか、エラーか）が未定義。

## 2. 現状の SPEC / UC

SPEC-LGX-006 §3 REQ.01 で mean pooling + L2 正規化を、REQ.02 で `embed --all` の全ノード生成を規定しているが、**入力テキストが空・空白のみの場合の挙動**は未定義。REQ.09 はサブノード入力を `content[range.0..range.1]` とするが、その範囲が空文字列になるケースの扱いも記述がない。

## 3. 期待される情報

SPEC または関連 REQ に追加されるべき記述:

- 空/空白のみテキストに対し (a) 生成をスキップする、(b) ゼロベクトルを格納する、(c) Error として計上し当該ノードのみ失敗扱いとする、のいずれを採るか
- mean pooling の分母が 0 になる場合の数値安定性の方針（GAP-LGX-104 と連携）
- スキップ採用時、当該ノードの drift 検出（REQ.05）・類似度計算（REQ.04）からの除外の有無

## 4. 影響範囲

この GAP がクローズされないと以下に影響:

- UC-LGX-007（embedding 生成）: 空ノードを含むグラフでの基本フロー事後条件が確定しない
- 下流 DD / TS: 空入力時のベクトル格納仕様・テストケースの期待値が書けない
- 数値安定性として GAP-LGX-104（ゼロベクトル cosine）と依存関係

## 5. 解決経緯（クローズ時に記入）

SPEC-LGX-006 v0.7.0（人間裁定 fix・承認 2026-06-10）: REQ.02 に空テキスト（正規化後 0 文字・空 content_range 含む）の skip + skipped 計上 + 集約 Warning を確定。ゼロベクトル非格納（0 除算の構造的回避）。skip ノードは未生成状態（REQ.05 の 3 状態）として drift/類似度から除外。

## 6. 関連 ADR

数値安定性ポリシー（空入力 + ゼロベクトル cosine）を横断する判断のため ADR 候補。GAP-LGX-104 と合わせて 1 件の ADR で扱う可能性あり。
