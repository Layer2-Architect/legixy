# GAP-LGX-235: UC-LGX-006 が graph.toml 不在・破損・パース不能の失敗パスを列挙していない

**親 TP**: TP-LGX-016
**観点**: 2.3 EF1（各ステップでの失敗パスが定義されているか）
**ステータス**: closed (2026-06-13)
**起票日**: 2026-06-13
**分類（暫定）**: WEAK（親 SPEC + LEGIXY-SPEC-001 で解決済。UC への明示反映は任意の可能性）

## 1. 観点

UC レベル観点「各ステップでの失敗パスが定義されているか」を UC-LGX-006 の事前条件「graph.toml が存在する」にぶつけた。事前条件が満たされない場合の失敗パスが UC フローに列挙されているかを問う。

## 2. 現状の UC / SPEC

- UC-LGX-006 の事前条件は「graph.toml が存在する」「起点となる成果物 ID が指定される」の 2 点のみ。
- graph.toml が**存在しない**場合の失敗パス、**破損・パース不能**（TOML 構文エラー等）の場合の失敗パスは、代替フロー / 例外フローのいずれにも記述されていない。
- 振る舞いは LEGIXY-SPEC-001 §4（エンジン機能）の実行前提 + SPEC-LGX-004.REQ.04（graph.toml 破損=exit 1）から導出可能。
- UC-LGX-001 の GAP-LGX-189（破損パス失敗フロー未列挙）と同型の問題。

つまり振る舞いは親 SPEC で確定済だが、UC-006 のフロー記述（失敗パス列挙）には反映されていない。

## 3. 推奨対応（人間裁定）

以下のいずれか:

- **(A) UC へ追記**: 代替フロー「graph.toml が存在しない場合、ERROR を報告して exit 1 で終了する」および「graph.toml が存在するが破損・パース不能の場合、ERROR を報告して exit 1 で終了する」を追加し、SPEC-LGX-004.REQ.04 を参照。→ UC のフロー完全性が向上。
- **(B) drop（委譲容認）**: 事前条件「graph.toml が存在する」は UC の前提として記述しており、前提が崩れた場合の失敗は SPEC-LGX-004（graph.toml 破損=exit 1）へ委譲する設計と認める。→ UC は変更しない。UC-001 の GAP-LGX-189 と同一裁定で一括処理可能。

WEAK 候補（UC フロー記述の粒度方針に依存）。GAP-LGX-189 と同種のため、同一裁定を推奨。

## 4. 影響範囲

- close されないと TP-LGX-016 が green にならず、UC-006 起点の下流（RBA 以降）に進めない。
- 振る舞い自体は SPEC-LGX-004/LEGIXY-SPEC-001 で確定済のため、下流実装の正しさには影響しない（記述完全性の問題）。
- UC-LGX-001 GAP-LGX-189 と同型。両者を一括裁定することで、UC フロー記述における事前条件失敗パスの扱いについて方針を統一できる。

## 5. 解消（2026-06-13）

敵対的精査裁定: **方針B（委譲容認）**。本観点は親 SPEC（対応 TP[SPEC] green 確定済）が回答しており、UC フローへの明示列挙は任意。GREEN-by-delegation として close。詳細: docs/spec-change-proposals/2026-06-13_uc-gap-adversarial-review.md §F。
