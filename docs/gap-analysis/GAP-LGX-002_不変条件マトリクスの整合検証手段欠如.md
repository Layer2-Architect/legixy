# GAP-LGX-002: umbrella §4.1 マトリクスと下位 SPEC §4 サブマトリクスの整合が保証・検証されていない

**親 TP**: TP-LGX-001
**観点出典**: TP-LGX-001 §2.2 観点 8（不変条件 × SPEC 責任マトリクスの完全性 — umbrella と下位 SPEC サブマトリクスの無矛盾性）
**ステータス**: closed (2026-06-10)
**起票日**: 2026-06-08
**severity**: minor (verification: low-value, 人間判断で drop 可)
**敵対的精査メモ（2026-06-09）**: §4.3 が整合性維持を「人間査読で維持（自動検証ツールは将来課題）」と**明示的に委譲・宣言**しており、観点 8 が問う「整合の保証手段」自体は（人間査読という手段で）回答済みに近い。残る要求（正準ソースの明示・同一コミット更新規律）は umbrella レベルの低価値な統治細目で、GAP-LGX-004（変更ポリシー欠如）の規律要件と内容が重複する。GENUINE ではなく WEAK_OR_PADDED と再判定。同期規律は GAP-LGX-004 側で扱うのが適切。

## 1. 観点

umbrella §4.1 の 28 行 × 10 列マトリクスと、各下位 SPEC（002〜010）§4 が掲載する不変条件サブマトリクスとの間に、実装 / 検証 / 関連 / - の食い違いが無いことが保証され、かつ検証されているか。

## 2. 現状の SPEC / UC

SPEC-LGX-001 §4.3 は「各下位 SPEC（002〜010）の §4 にも、本 SPEC で担当する不変条件のサブセット表を掲載する。**本表と下位 SPEC 表の整合性は人間査読で維持する（自動検証ツールは将来課題）**」と記述している。

すなわち umbrella は (a) 整合性維持の手段を「人間査読」のみに委ね、(b) 自動検証は将来課題として明示的に未実装とし、(c) **現時点で両者が実際に整合しているという確認結果（査読済みであること）を主張していない**。28 行 × 10 列 = 280 セルの二重管理に対し、不一致が混入しても検出する仕組みが無く、umbrella マトリクスが正準であるという前提も明文化されていない。

## 3. 期待される情報

SPEC に追加されるべき記述:

- 二重管理されるマトリクスの **正準ソース**（umbrella §4.1 を正、下位 SPEC §4 を派生ビューとする、等）の明示。どちらが矛盾時の権威かを規定する。
- 「現バージョン時点で umbrella §4.1 と下位 SPEC §4 サブマトリクスは査読により整合確認済」である旨の確認記録（査読日・査読者・対象列）、または未確認なら未確認である旨の明示。
- 自動検証を将来課題とするなら、その間の不一致リスクを抑える運用規律（例: 不変条件責任を変更する SPEC 改訂時は umbrella §4.1 と当該下位 SPEC §4 を同一コミットで更新する、というハードルール化）。

## 4. 影響範囲

この GAP がクローズされないと以下に影響:

- 検証フェーズ（SPEC-LGX-004）と各実装 SPEC: ある不変条件の実装/検証 owner が umbrella と下位 SPEC で食い違ったまま下流に伝播すると、TS/TC でどの SPEC を根拠に検証を書くべきかが曖昧になる。
- 下流の RBA 以降: 不変条件の責任所在が成果物追跡の根拠であるため、不整合は誤った親参照を誘発しうる。
- 他の GAP との依存: GAP-LGX-003（検証 owner 要否ポリシー）と同じ §4 マトリクスの統治に関わる。

## 5. 解決経緯（クローズ時に記入）

SPEC-LGX-001 v0.7.1（人間裁定 fix・承認 2026-06-10）: §4.3 に査読確認記録（2026-06-10 整合確認済）と §7.3（正準ソース宣言・同一コミット規律 — GAP-004 で新設済）への参照を追記。同期規律の実体は §7.3 が吸収済み。

## 6. 関連 ADR

- 「umbrella §4.1 を正準ソースとし下位 SPEC §4 を派生ビューとする」設計判断を採る場合、ADR-LGX-NNN を起票（マトリクスの単一ソース化は architectural 判断）。
