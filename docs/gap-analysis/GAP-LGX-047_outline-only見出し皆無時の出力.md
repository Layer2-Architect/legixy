# GAP-LGX-047: `--outline-only` で対象 artifact に h1〜h3 見出しが 1 つも無い場合の出力

**親 TP**: TP-LGX-003
**観点出典**: TP-LGX-003 §2.8 観点 L-02
**ステータス**: closed (2026-06-10)
**起票日**: 2026-06-08
**重大度**: minor (verification: low-value, 人間判断で drop 可)

> **敵対的精査（2026-06-09）所見**: 主たる帰結は REQ.15 から導出可能。「本文（見出し以外のテキスト）は出力に含めない」が明示済みのため、h1〜h3 が 0 個なら当該 artifact の body は空になる、と一意に読める。残るのは「artifact の枠を残して body 空とするか／artifact ごと省略するか」の DD レベルのフォーマット選択のみで、これは REQ.10（セクション構成は granularity・件数非依存で固定 = artifact 枠は維持、body 空）から推定可能。決定論（CACHE-INV-1）も空 body のフォーマットを固定すれば満たせる。意思決定価値は低く、人間判断で drop 可。

## 1. 観点

`--outline-only` は各 upstream artifact の本文を ATX 見出し（h1〜h3）の階層リストに置換するが、対象 artifact に h1〜h3 見出しが 1 つも存在しない場合（見出し皆無の本文、h4 以降のみ、`#abc` 等のスペース無し見出しのみ）の出力が定義されていない。空の本文になるのか、artifact 自体が省略されるのか、プレースホルダが入るのかが不明。

## 2. 現状の SPEC / UC

- SPEC-LGX-003 REQ.15 は次を規定する: 各見出しを `- {title}` 行として出力、レベルに応じてインデント、`####` 以降（h4 以降）は対象外、見出し記号と本文の間にスペースが無い行（`#abc` 等）は対象外、subnode 粒度では body を anchor 文字列のみに置換。
- 規定されていないのは、**対象範囲となる h1〜h3 見出しが 0 個**だったときの出力。REQ.15 は「本文（見出し以外のテキスト）は出力に含めない」とするため、見出しが皆無なら当該 artifact の body は完全に空になると読めるが、その場合に:
  - artifact のヘッダ（ノード ID 等の枠）だけ残して body 空とするのか、
  - artifact ごと upstream から省略するのか、
  が明示されていない。
- これは REQ.10（セクション配置順序）・REQ.14（バイト決定論）にも影響する（空 body のフォーマットが固定でないとバイト列が揺れる）。

## 3. 期待される情報

SPEC-LGX-003 REQ.15 に以下を追加:

- h1〜h3 見出しが 0 個の artifact の出力規約: (a) artifact の枠（ID 見出し等）を残し body を空（または固定プレースホルダ）にする、(b) artifact ごと upstream から省略する、のいずれを正準とするか。
- 空 body 時の正確なフォーマット（改行・空白を含む）を、CACHE-INV-1（バイト決定論）を満たす形で固定。
- subnode 粒度との組合せ（anchor のみ置換）で anchor も無い縮退ケースがあれば併せて規定。

## 4. 影響範囲

この GAP がクローズされないと以下に影響:

- UC-LGX-004（粒度別テスト拡充）: 見出し皆無 artifact の outline 化フローが書けない。
- 下流 DD: outline 変換処理（見出し抽出 → 0 件時の分岐）の設計が確定しない。
- TS/TC: TS-LGX-002 §15 T-CC-OUTLINE-001 の見出し皆無ケースの期待値が定まらない。
- CACHE-INV-1: 空 body のフォーマットが未固定だとバイト決定論が崩れる。

## 5. 解決経緯（クローズ時に記入）

SPEC-LGX-003 v0.7.1（人間裁定 fix・承認 2026-06-10）: REQ.15 に見出し皆無時の出力規約を追記 — artifact 枠を維持し body 空（省略しない、REQ.10 整合）、プレースホルダなし、空 body の固定フォーマットは CACHE-INV-1 を満たす形で DD 確定、anchor 無し縮退も同規約。

## 6. 関連 ADR

該当なし（出力規約の明確化レベル）。
