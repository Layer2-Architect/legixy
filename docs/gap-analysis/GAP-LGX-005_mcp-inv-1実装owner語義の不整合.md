# GAP-LGX-005: MCP-INV-1 の「実装」owner に umbrella 自身が指定されており凡例語義と整合しない

**親 TP**: TP-LGX-001
**観点出典**: TP-LGX-001 §2.3 観点 11（アーキテクチャ境界の一貫性 — MCP-INV-1 の実装 owner としての umbrella 指定）
**ステータス**: closed (2026-06-10)
**起票日**: 2026-06-08
**severity**: minor (verification: low-value, 人間判断で drop 可)
**敵対的精査メモ（2026-06-09）**: 事実関係は正しい（§4.1 で 001 列が全 28 行中 MCP-INV-1 のみ「実装」、他 27 行は「関連」）。ただし MCP-INV-1 の実機構 owner が SPEC-LGX-009 であることは §4.2 および SPEC-LGX-009 §4 表（役割=実装, 対応要求 REQ.02）で明示済みであり、下流の実装先特定に誤りは生じない。1 セルの凡例語義不整合に過ぎず決定関連性が低い。GENUINE ではなく WEAK_OR_PADDED と再判定（人間判断で 1 セル訂正 or drop）。

## 1. 観点

§4.1 マトリクスで MCP-INV-1（Agent Surface 限定）の「実装」owner として SPEC-LGX-001 自身（umbrella、001 列）が指定されている整合性。umbrella は具体的機構を持たず方針を宣言するのみであり、凡例「実装＝具体的機構を規定する」と語義が整合するか。

## 2. 現状の SPEC / UC

SPEC-LGX-001 §4.1 凡例は「**実装**: 当該 SPEC の REQ が不変条件を実現する具体的機構を規定する」「**関連**: 直接の実装・検証はないが、REQ が当該不変条件に関与する（継承宣言・前提・補強等）」と定義している。

MCP-INV-1 行を見ると 001 列が「実装」となっている（他に 003/005/009/010 も実装）。しかし umbrella の REQ.08 は「Agent Surface は MCP 3 ツールのみ」「承認操作は Admin Surface にのみ存在し Agent から呼び出してはならない」という **方針・制約の宣言** であって、3 ツール限定を実現する具体的機構（MCP サーバの公開ツール定義・引数転送）は SPEC-LGX-009 が規定する。したがって凡例の語義に厳密に照らすと、umbrella の MCP-INV-1 への関与は「実装」ではなく「**関連**（方針の宣言・継承宣言）」に該当する。

なお他の全 27 行で umbrella（001 列）は一貫して「関連」とされており、MCP-INV-1 の 001 列だけが「実装」となっている点で、umbrella の役割（方針宣言に徹する非実装層）として表内一貫性も欠く。

## 3. 期待される情報

SPEC に追加されるべき記述（どちらかの方向で語義を一致させる）:

- 案 A: MCP-INV-1 行の 001 列を「実装」→「関連」に訂正し、umbrella が方針宣言層であることと表内一貫性を回復する（実装機構は 009 が担う旨は §4.2 で既述）。
- 案 B: 凡例を改訂し「umbrella REQ が不変条件を直接の規範制約として規定する場合も実装に含む」と定義を拡張する。ただしこの場合、他の不変条件で umbrella が規範を述べている箇所（REQ.06 不変条件継承宣言等）との一貫性を再点検する必要がある。

いずれを採るにせよ、§4.1 の「実装」と「関連」の境界を umbrella レベルで一意にする。

## 4. 影響範囲

この GAP がクローズされないと以下に影響:

- 限定的（minor）: MCP-INV-1 の実際の実装機構が SPEC-LGX-009 にあることは §4.2 で明示されているため、下流の実装先の特定に致命的な誤りは生じにくい。
- ただし §4.1 の凡例語義が緩いまま放置されると、GAP-LGX-002（マトリクス整合検証）/ GAP-LGX-003（検証 owner 要否）の判定基準も曖昧になり、マトリクス全体の信頼度が下がる。

## 5. 解決経緯（クローズ時に記入）

SPEC-LGX-001 v0.7.1（人間裁定 fix・承認 2026-06-10、案A）: §4.1 MCP-INV-1 行の 001 列を「実装」→「関連」に訂正。umbrella は方針宣言層であり実装機構は SPEC-LGX-009（§4.2 既述）。001 列の表内一貫性（全行「関連」）を回復。
