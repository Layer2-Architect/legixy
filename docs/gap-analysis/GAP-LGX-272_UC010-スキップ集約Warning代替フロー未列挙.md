# GAP-LGX-272: UC-LGX-010 がスキップ発生時の集約 Warning を代替フローに列挙していない

**親 TP**: TP-LGX-020
**観点**: 2.2 AF1（分岐網羅）/ 2.6 R6（スキップ時集約 Warning の UC 表現）
**ステータス**: closed (2026-06-13)
**起票日**: 2026-06-13
**分類（暫定）**: WEAK（親 SPEC で規定済。UC への列挙は任意の可能性）

## 1. 観点

UC レベル観点「分岐条件が網羅されているか」を UC-LGX-010 の代替フローにぶつけた。SPEC-LGX-010 REQ.04 は端点 embedding 不在・次元不一致によるスキップ発生時の「集約 Warning 1 件（スキップ件数と代表理由）を stderr に出力」を明示しているが、この分岐が UC の代替フローに列挙されていない。

## 2. 現状の UC / SPEC

- UC-LGX-010 の代替フローは `2a`（embeddings テーブルが空の場合）/ `3a`（compute 失敗）の 2 件のみ。
- SPEC-LGX-010 REQ.04 は「端点 embedding 不在・次元不一致によりスキップが発生した場合、**集約 Warning 1 件**（スキップ件数と代表理由）を stderr に出力する」と規定する（v3 差分: v3 は無言スキップ）。
- SPEC-LGX-010 REQ.09 は非有限スコア（NaN/±Inf）のペアについても「skip + 集約 Warning 1 件」と規定する。
- これらスキップケースは正常計算フロー内で発生する部分スキップであり、compute 全体失敗（3a）とは異なる。UC の代替フロー分岐としての列挙がない。

## 3. 推奨対応（人間裁定）

以下のいずれか:

- **(A) UC へ追記**: 代替フロー `3b`「端点 embedding 不在・次元不一致・非有限スコアのペアが存在する場合: 当該ペアをスキップし、集約 Warning 1 件（件数 + 代表理由）を stderr に出力する。計算を継続し基本フロー Step4/5 へ進む。exit 0」を追加し、SPEC-LGX-010 REQ.04/REQ.09 を参照。→ 部分スキップ分岐が UC で観察可能化。
- **(B) drop（委譲容認）**: 部分スキップは正常計算フロー内の SPEC-LGX-006 REQ.11 / SPEC-LGX-010 REQ.04 の動作であり、UC の代替フロー列挙粒度（主要分岐のみ）に含めない設計と認める。計算エンジン挙動の詳細は親 SPEC + TP-LGX-010 へ委譲。→ UC は変更しない。

WEAK 候補（親 SPEC で規定済。スキップは正常実行経路内の動作であり、UC 代替フローに挙げるべき「主要分岐」かどうかは粒度方針に依存）。

## 4. 影響範囲

- close されないと TP-LGX-020 が green にならず、UC-010 起点の下流に進めない。
- 振る舞い自体は SPEC-LGX-010 REQ.04/REQ.09 で確定済のため、下流実装の正しさには影響しない（記述完全性の問題）。
- GAP-LGX-189 / GAP-LGX-190（UC-001 系）と同種の問題であり、一括裁定方針が可能。

敵対的精査裁定: **方針B（委譲容認）**。本観点は親 SPEC（対応 TP[SPEC] green 確定済）が回答しており、UC フローへの明示列挙は任意。GREEN-by-delegation として close。詳細: docs/spec-change-proposals/2026-06-13_uc-gap-adversarial-review.md §F。
