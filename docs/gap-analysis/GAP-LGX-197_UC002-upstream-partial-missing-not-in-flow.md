# GAP-LGX-197: 上流連鎖途中欠損（REQ.20 部分成功）が Step3 の継続挙動として未反映

**親 TP**: TP-LGX-012
**観点**: §2.3 EF4
**ステータス**: closed (2026-06-13)
**起票日**: 2026-06-13
**分類（暫定）**: WEAK

## 1. 観点

SPEC-LGX-003.REQ.20(2) は「上流連鎖途中の欠損（エッジ先ノードのファイル不在等）は欠損ノードを飛ばして残りの上流を返す部分成功・exit 0。欠損は出力内に決定論的に記録する」と規定する。UC-LGX-002 の基本フロー Step3「システムが成果物 ID から有向グラフを逆方向に走査し、上流成果物を収集する」はこの欠損時の継続挙動（ノードスキップ・部分収集継続・欠損の決定論的記録）がフロー記述に現れない。Step3 で一部ノードが欠損した場合に処理が継続されることが観察不能。

## 2. 現状の UC / SPEC

- **UC-LGX-002 基本フロー Step3**: 「システムが成果物 ID から有向グラフを逆方向に走査し、上流成果物を収集する」（欠損時の挙動未記述）
- **SPEC-LGX-003.REQ.20 (2)**: 「上流連鎖途中の欠損（エッジ先ノードのファイル不在等）: 欠損ノードを**飛ばして残りの上流を返す部分成功・exit 0**。欠損は出力内に**決定論的に記録**する」
- **TP-LGX-003 E-05（GAP-LGX-043 closed）**: SPEC レベルでは解消済み

## 3. 推奨対応（人間裁定）

**(A) UC-LGX-002 に代替フロー 3-A を追記する**
「3-A. 上流走査中に一部ノードのファイルが存在しない場合、当該ノードを飛ばして残りの上流成果物の収集を継続する（部分成功）。欠損情報は返却内容に決定論的に記録する（SPEC-LGX-003.REQ.20）」として Step3 の部分失敗継続挙動を明示する。

**(B) drop（委譲容認）**
SPEC-LGX-003.REQ.20 が部分成功の挙動を確定しており、UC フロー記述に反映せずとも設計上の空白はない。Step3 の「上流成果物を収集する」は部分収集を排除していないため、SPEC 委譲で整合とみなす。

## 4. 影響範囲

- UC-LGX-002 代替フロー（追記の場合）
- TP-LGX-012 EF4（解消後 GREEN 化）
