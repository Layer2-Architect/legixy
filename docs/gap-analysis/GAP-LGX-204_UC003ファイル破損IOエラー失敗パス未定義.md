# GAP-LGX-204: UC-LGX-003 代替フロー 2a がファイル破損・IO エラーを扱っていない

**親 TP**: TP-LGX-013
**観点**: §2.3 EF1
**ステータス**: closed (2026-06-13)
**起票日**: 2026-06-13
**分類（暫定）**: WEAK

## 1. 観点

UC-LGX-003 代替フロー 2a は「ドキュメントファイルが存在しない場合、そのノードのサブノード抽出はスキップする」とファイル「不在」のみを扱っている。ファイルが存在するが「破損・UTF-8 不正・IO エラー・パーミッション拒否」の場合の失敗パスが UC フローに定義されていない。

## 2. 現状の UC / SPEC

- UC-LGX-003 代替フロー 2a: 「ドキュメントファイルが存在しない場合」——不在のみを扱い、存在するが読み取れない場合は未定義。
- SPEC-LGX-002.REQ.10: 「graph.toml パーサおよび Markdown 見出し抽出は、大きすぎる Markdown ファイル、深いネスト見出し、不正な TOML、project_root 外への参照パスでクラッシュしてはならない。メモリ不足等の OS 起因の失敗は通常のエラー経路（exit 1）で報告する」——クラッシュ禁止と exit 1 報告を規定するが、UC のフロー記述（失敗パス列挙）への反映はない。
- TP-LGX-002 G-EH-1（GREEN 確立済）: 不正 TOML 時のクラッシュ禁止を REQ.10 + NFR SEC.03 へ委譲して確立済み。

## 3. 推奨対応（人間裁定）

(A) **UC へ追記案**: 代替フロー 2a を拡張し「ドキュメントファイルが存在しない、または読み取り不能（IO エラー・パーミッション拒否・UTF-8 不正等）の場合、そのノードのサブノード抽出をスキップし、SPEC-LGX-002.REQ.10 に従い通常エラー経路で報告する（exit 1）」と明示する。

(B) **drop（委譲容認）案**: SPEC-LGX-002.REQ.10（クラッシュ禁止 + exit 1 報告）が当該失敗パスを所有しており、UC フロー記述への明示は冗長として drop する。

## 4. 影響範囲

- UC-LGX-003 §代替フロー 2a
- 影響度は低。SPEC-LGX-002.REQ.10 が答えており、下流成果物（RBA/DD/TS）への波及は軽微。TP-LGX-011 GAP-LGX-189 と類似パターン（破損失敗パス未列挙）。

## 5. 解消（2026-06-13）

敵対的精査裁定: **方針B（委譲容認）**。本観点は親 SPEC（対応 TP[SPEC] green 確定済）が回答しており、UC フローへの明示列挙は任意。GREEN-by-delegation として close。詳細: docs/spec-change-proposals/2026-06-13_uc-gap-adversarial-review.md §F。
